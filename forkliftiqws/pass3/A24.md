### A24-1
**Severity:** HIGH
**File:** `src/main/java/com/journaldev/spring/jdbc/model/User.java`
**Location:** Class declaration, line 13
**Finding:** The `User` class has no class-level Javadoc comment. This is a core domain model representing an authenticated system user with security-sensitive fields (`password`, `roles`, `active`). The absence of class-level documentation leaves the intended lifecycle, ownership, and security constraints entirely undocumented.

---

### A24-2
**Severity:** HIGH
**File:** `src/main/java/com/journaldev/spring/jdbc/model/User.java`
**Location:** `addRoles(Roles role)`, line 41
**Finding:** The only explicitly written public method in `User.java`, `addRoles(Roles role)`, has no Javadoc. There is no `@param` tag documenting the `role` parameter, no description of the method's behavior (add to internal `HashSet`), no indication of null-safety, and no note on whether duplicate roles are silently ignored (which they are, due to `HashSet` semantics). This behavior is non-obvious and unspecified.
