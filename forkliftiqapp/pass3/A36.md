#### A36-2: Both public constructors lack Javadoc [INFO]
- File: `BaseImageDownloader.java`, Lines: 73, 77
- Description: Neither constructor has a Javadoc comment. The two-argument constructor at line 77 accepts `connectTimeout` and `readTimeout` parameters (in milliseconds, consistent with the constant names) but there is no documentation describing the parameters, their units, or the fact that `context.getApplicationContext()` is stored (not the passed context directly). The one-argument constructor at line 73 delegates to the two-argument form using the default timeout constants; this delegation is undocumented.

#### A36-3: `getStream` override has no Javadoc [INFO]
- File: `BaseImageDownloader.java`, Line: 84
- Description: `getStream` is an `@Override` of the `ImageDownloader` interface method. While Javadoc inheritance from the interface is implied by tooling, no `@inheritDoc` tag or supplementary documentation is present. The implementation dispatches across seven URI schemes; subclass authors and maintainers have no inline guidance about the dispatch logic or the fallback to `getStreamFromOtherSource` for unknown schemes.

#### A36-4: `shouldBeProcessed` has a bare `@throws IOException` tag with no description [LOW]
- File: `BaseImageDownloader.java`, Line: 143
- Description: The Javadoc for `shouldBeProcessed` (lines 138-143) includes `@throws IOException` with no explanatory text following the tag. A bare `@throws` tag generates an empty entry in generated API docs and provides no guidance to callers about when the exception is raised or what conditions trigger it. The implementation at line 145 only reads `conn.getResponseCode()`, which can itself throw `IOException` if the connection is not yet connected or the response cannot be read; that scenario is left entirely unexplained.

#### A36-5: `createConnection` `@throws IOException` description is copied inaccurately from `getStreamFromNetwork` [LOW]
- File: `BaseImageDownloader.java`, Lines: 155-156
- Description: The `@throws IOException` for `createConnection` reads: "if some I/O error occurs during network request or if no InputStream could be created for URL." The method returns an `HttpURLConnection`, not an `InputStream`; the phrase "if no InputStream could be created for URL" is copy-pasted from `getStreamFromNetwork`'s Javadoc and is factually incorrect in this context. `createConnection` never creates an `InputStream`, so the second clause of the `@throws` description is misleading to callers and subclass implementors.

#### A36-6: `getContactPhotoStream` has no Javadoc [INFO]
- File: `BaseImageDownloader.java`, Line: 229
- Description: The protected method `getContactPhotoStream(Uri uri)` has no Javadoc. It is called from `getStreamFromContent` when the URI starts with the contacts prefix. The method applies API-level branching (`Build.VERSION_CODES.ICE_CREAM_SANDWICH`) to choose between the two-argument and three-argument forms of `openContactPhotoInputStream`. The API-level requirement, the meaning of the `true` argument (requesting the full-size photo), and the possible `null` return value are all undocumented.

#### A36-7: `getStreamFromOtherSource` Javadoc contains a spelling error [INFO]
- File: `BaseImageDownloader.java`, Line: 267
- Description: The Javadoc comment says "Should be overriden by successors". The correct past-participle spelling is "overridden" (double 'd'). While functionally harmless, the error appears in the published API documentation for a method whose primary purpose is to guide subclass authors, making the misspelling particularly visible.
