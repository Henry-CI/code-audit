### P13-F02: No Authentication Guard on GPS Data API Endpoints
**Description:** `gps/unit.gps.jsp` and `gps/unit.gps_nd.jsp` are data endpoints that return real-time GPS positions and historical GPS position data for vehicles. Neither file includes `<%@ include file="../sess/Expire.jsp" %>` or any other session validation. Any unauthenticated request with a valid (or guessed) customer code (`ccd`), site code (`site`), and department code (`dep`) will receive vehicle location data. The display pages (`gpsWhereIAm.jsp`, `speedZones.jsp`) do include `Expire.jsp`, but the data endpoints that those pages call via AJAX do not — meaning a direct HTTP request to the data endpoint bypasses the session check entirely.
**Severity:** CRITICAL
**Checklist item:** Section 2 — Authentication and Authorization; Section 5 — Data Exposure and Privacy (GPS/location data)
**Evidence:**
```jsp
// gps/unit.gps.jsp — lines 1-4, no Expire.jsp include:
<%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%>

<%@ page errorPage="../home/ExceptionHandler.jsp" %>
<jsp:useBean class="com.torrent.surat.fms6.reports.Databean_report" id="filter" scope="request"/>

// Compare with gpsWhereIAm.jsp lines 1-2 which does include the guard:

<%@ include file="../sess/Expire.jsp" %>
```
**File:** `gps/unit.gps.jsp`, `gps/unit.gps_nd.jsp`

### P13-F04: IDOR — Customer Code Supplied by Request Parameter with No Session-Based Ownership Verification
**Description:** All five data-serving endpoints (`unit.gps.jsp`, `unit.gps_nd.jsp`, `gpsZonesData.jsp`, `Copy of gpsZonesData.jsp`, `setGpsZones.jsp`) accept the customer identifier (`ccd`) as a plain URL/form request parameter. There is no visible check at the JSP layer that compares the requested customer code against the authenticated user's customer from the session. An authenticated user could substitute another customer's `ccd` value in the URL and retrieve (or overwrite) that customer's GPS data and geofence zones. This is an Insecure Direct Object Reference affecting vehicle location data and geofence configurations across all customers.
**Severity:** CRITICAL
**Checklist item:** Section 2 — Authorization and multi-tenancy (IDOR)
**Evidence:**
```jsp
// gps/unit.gps.jsp lines 3-6:

String st_cd=request.getParameter("site")==null?"0":request.getParameter("site");

// No: session.getAttribute("access_cust") comparison against cust_cd

// gps/setGpsZones.jsp lines 8-9:
String st_cd=request.getParameter("site");
String cust_cd=request.getParameter("ccd");
// No session check; cust_cd passed directly to bean for write operation
```
**File:** `gps/unit.gps.jsp`, `gps/unit.gps_nd.jsp`, `gps/gpsZonesData.jsp`, `gps/Copy of gpsZonesData.jsp`, `gps/setGpsZones.jsp`
