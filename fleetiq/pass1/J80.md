### J80-F01: SQL injection via set_gp_cd in Fetch_users()
**Description:** The `set_gp_cd` field is populated by callers (JSP pages via `setSet_gp_cd()`) and is concatenated directly into a SQL query string used with a `Statement` (not a `PreparedStatement`). Any value for `set_gp_cd` originating from a request parameter is injected verbatim into the query. An attacker controlling this parameter can retrieve arbitrary data from the database or cause other SQL-level harm.
**Severity:** HIGH

### J80-F06: No customer/tenant scoping on any data query — multi-tenancy isolation failure
**Description:** Neither `LogicBean_filter`, `LogicBean_filter1`, nor `Menu_Bean` apply any customer ID or tenant ID constraint to their SQL queries. The branch name (`multi-customer-sync-to-master`) and the checklist explicitly identify multi-tenant data isolation as the highest-risk area. The `Fetch_users()` method retrieves ALL groups and ALL users from the database with no WHERE clause restricting to the authenticated user's customer. Similarly, `Fetch_Data()` retrieves driver records using only the user ID and date range — no customer scoping. A user authenticated to one customer can, by supplying another customer's user IDs, retrieve that customer's driver activity records.
**Severity:** CRITICAL

## Checklist Items With No Issues

The following checklist items were reviewed for these three files and no issues were found within the scope of the files read:

- **1. Secrets and Credentials:** No hardcoded database credentials, API keys, SMTP passwords, or connection strings are present in any of the three files. Database connection is obtained via JNDI `InitialContext` lookup using `RuntimeConf.penom_database` / `RuntimeConf.invoice_database` (references to a configuration class, not literal credentials). No credentials in comments.
- **3. Injection — Command injection:** No `Runtime.getRuntime().exec()` or `ProcessBuilder` usage present in any of the three files.
- **3. Injection — SSRF:** No `HttpURLConnection`, `HttpClient`, or outbound HTTP calls present in any of the three files.
- **3. Injection — File inclusion:** No file inclusion constructs present in these Java files (file inclusion applies to JSP files; these are Java bean classes).
- **6. Transport Security:** No transport-layer configuration present in these files; TLS and cookie security are configured at the Tomcat/web.xml layer, outside the scope of these files.
