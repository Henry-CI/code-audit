### P21-F01: Hardcoded Access Level Bypass in Firmware Pages (Authentication/Authorization Disabled)

**Description:** `frm_conf_firmware_display.jsp` and `frm_conf_firmware_upg.jsp` both hardcode the access control variables instead of reading them from the session. In both files the session-based access control code is commented out and replaced with literal values. This means anyone who can reach these pages is treated as access level 1 (highest privilege) regardless of who they actually are. The session user_cd check is also commented out and replaced with a hardcoded `"1"`. These pages control viewing of vehicle firmware versions and initiating firmware upgrades across all vehicles — highly sensitive operations.

**Severity:** CRITICAL

**Checklist item:** Section 2 — Authentication and Authorization

**Evidence:**
```java
// frm_conf_firmware_display.jsp lines 72–90:

String access_cust="0";

// frm_conf_firmware_upg.jsp lines 520–538:

String access_cust="0";
String access_site="0";
String access_dept="0";
...
//String ucd = ""+session.getAttribute("user_cd");
```

**File:** master/frm_conf_firmware_display.jsp (lines 72–90), master/frm_conf_firmware_upg.jsp (lines 520–538)

### P21-F02: Missing Session Expiry Check on Firmware Pages

**Description:** `frm_conf_firmware_display.jsp` and `frm_conf_firmware_upg.jsp` do not include the `../sess/Expire.jsp` session check that is present in all other files in this batch. Combined with the hardcoded access control (F01), these pages are completely unprotected from unauthenticated access. All other pages in this set include the session expire check at line 1; these two pages begin with a blank line then the page import, with no session check at all.
