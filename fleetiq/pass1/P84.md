### P84-F01: Cross-Site Scripting (XSS) - Unescaped User Input in HTML Context
**Description:** Multiple JSP files output user-controlled request parameters and database values directly into HTML without escaping, allowing attackers to inject malicious JavaScript. User-supplied values from request parameters (user_cd, loc_cd, dept_cd, model_cd, form_cd, st_dt, to_dt, etc.) and database-retrieved values (form_nm, customer names, site names, vehicle names) are rendered using `<%= ... %>` expressions without any encoding or sanitization.
**Severity:** HIGH
**Checklist item:** 3. Injection Vulnerabilities - XSS (Cross-Site Scripting)
**Evidence:**
```jsp
Line 435: <body bgcolor="white" onload="set('<%=user_cd %>','<%=loc_cd %>','<%=dept_cd %>','<%=model_cd %>','<%=fld_cd %>')">
```
