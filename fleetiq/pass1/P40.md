### P40-F01: Missing Session Expiry / Authentication Check in email_subscription_report_uk.jsp
**Description:** `email_subscription_report_uk.jsp` does not include `Expire2.jsp` (the session expiry and authentication check). All other nine files in this batch include `<%@ include file="../../../sess/Expire2.jsp" %>` as their authentication gate. The UK variant of the email subscription report omits this entirely, meaning the page's authentication/session-expiry logic is absent. A user with a stale or invalid session — or potentially any unauthenticated request that bypasses the outer container dispatcher — could reach this page without the check that would otherwise redirect them.
**Severity:** HIGH
**Checklist item:** Section 2 — Authentication and Authorization
**Evidence:**
```jsp

    ...
```
(No `<%@ include file="../../../sess/Expire2.jsp" %>` anywhere in the file — contrast with `email_subscription_report.jsp` line 5 which has it.)
**File:** pages/admin/reports/email_subscription_report_uk.jsp

### P40-F02: Reflected XSS via `sub` Parameter in Breadcrumb href (Multiple Files)
**Description:** All ten files render the `sub` request parameter directly into an unescaped `href` attribute in the breadcrumb navigation using `<%= sub %>`. There is no HTML encoding applied. An attacker can craft a URL with a malicious value for `sub` (e.g., `"><script>alert(1)</script>`) and the content will be injected into the page HTML. This is a stored-if-persisted or reflected XSS vulnerability depending on context. The pattern is identical across all files reviewed.
**Severity:** HIGH
**Checklist item:** Section 3 — XSS (Cross-Site Scripting)
**Evidence (representative — from driver_licence_expiry_report.jsp):**
```jsp
// Line 96 (scriptlet):
String sub = request.getParameter("sub")==null?"":request.getParameter("sub");
// Line 107 (HTML output):
<li><a href="../pages/admin.jsp?mnm=Reports&sub=<%=sub%>"> <fmt:message key="rpt.driverlicenceexpiryreport.title" /></a>
```
Same pattern at:
- driver_proficiency_report.jsp — no breadcrumb sub output (not present)
- email_subscription_report.jsp line 86
- email_subscription_report_uk.jsp line 82
- equipment_bypass_alert_report.jsp line 70
- gps_report.jsp line 142
- gps_route_report.jsp line 145
- gps_wifi_report.jsp line 144
- hour_count_report.jsp line 80

## Checklist Items With No Issues

The following checklist items were reviewed against the ten files and no issues were found within the scope of these files:

**Section 1 — Secrets and Credentials:** No hardcoded database credentials, API keys, SMTP passwords, or encryption keys found in any of the ten files. No `DriverManager.getConnection()` calls or similar JDBC setup. No commented-out connection strings with passwords.

**Section 2 — Authentication and Authorization (partial):** All nine files that include `Expire2.jsp` appear to have an authentication gate. Customer/tenant scoping is delegated to the `Databean_report` and `Databean_dyn_reports` beans via the `access_level`, `access_cust`, `access_site`, and `access_dept` session attributes — these are read from the session (server-side), not from request parameters alone. The filter beans receive these values and are expected to enforce them in the data layer. No admin-only operations were identified in these ten report display files (the delete operation in the email subscription files is user-scoped, not admin).

**Section 3 — File Inclusion:** No dynamic file inclusion found. All `<%@ include %>` and `<jsp:include>` directives use static, hardcoded paths. No user-controlled file path parameters were found.

**Section 3 — Command Injection:** No `Runtime.getRuntime().exec()` or `ProcessBuilder` calls found in any of the ten files.

**Section 3 — SSRF:** No `HttpURLConnection`, `HttpClient`, or similar HTTP client calls found in the JSP files themselves. The AJAX calls in `gps_report.jsp`, `gps_route_report.jsp`, and `gps_wifi_report.jsp` are client-side JavaScript making requests to the same application origin (`../master/get_cust_vehicle.jsp`), not to user-controlled external URLs.

**Section 6 — Transport Security:** No explicit HTTP connector configuration found within these JSP files (that is a `server.xml`/`context.xml` concern). Session cookie configuration is not controlled from within these JSP files.

**Section 7 — Dependency Vulnerabilities (JAR/Maven):** Not applicable to JSP view files. CDN-loaded third-party JavaScript is covered under P40-F10.
