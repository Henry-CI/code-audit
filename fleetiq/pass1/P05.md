## Findings

### P05-F01: Reflected XSS — Broadcast Message Text Output Unescaped
**Description:** In `print_broadcastmsg.jsp`, the broadcast message text (retrieved from the database) is rendered unescaped: `<%=broadcastmsgBean.getText() %>`. Since broadcast messages are composed by fleet administrators or via automated systems and stored in the database, if any administrator enters HTML or JavaScript content in a message text, it will be rendered as active HTML in the print view for any user who views that message.
**Severity:** MEDIUM
**Checklist item:** 3. Injection Vulnerabilities — XSS (stored)
**Evidence:**
```jsp
<td><%=broadcastmsgBean.getText() %></td>
```
**File:** au_print/print_broadcastmsg.jsp
**Line(s):** ~74 (within table data loop)

### P05-F02: Unescaped Database-Sourced Values Throughout Print Reports — XSS
**Description:** All print JSP files in this partition render database-sourced values (vehicle names, driver names, model names, customer names, timestamps) directly with `<%= ... %>` without HTML escaping. While these are typically trusted database values, they may originate from user-supplied input (vehicle name, driver name fields entered by administrators) and must be escaped.
**Severity:** MEDIUM
**Checklist item:** 3. Injection Vulnerabilities — XSS
**Evidence:**
```jsp
<td><%=vunit_name.get(i) %></td>
<td><%=vdriver_name.get(i) %></td>
<td><%=broadcastmsgBean.getUnit() %></td>
<td><%=driv_nm %></td>
```
**File:** au_print/print_battery.jsp, print_broadcastmsg.jsp, print_curr_driv_report.jsp, print_daily_veh_summary.jsp, and others
**Line(s):** Multiple throughout

### P05-F03: Missing CSRF Protection
**Description:** Print report pages are invoked via GET/POST from parent report pages without CSRF token protection, consistent with the application-wide pattern.
**Severity:** HIGH
**Checklist item:** 4. Session and CSRF
**Evidence:** No CSRF tokens in any of the 10 files.
**File:** All files in this partition
**Line(s):** N/A — absence throughout

### P05-F04: Hardcoded HTTP URLs for Images (Mixed Content)
**Description:** `print_battery.jsp` and `print_hourmeter_exception.jsp` contain hardcoded `http://fleetfocus.lindemh.com.au/...` image URLs. These are plain HTTP and constitute mixed content if the application runs over HTTPS.
**Severity:** LOW
**Checklist item:** 6. Transport Security
**Evidence:**
```html
<img src="http://fleetfocus.lindemh.com.au/fms/images/linde/linde_long_logo.gif" .../>
```
**File:** au_print/print_battery.jsp, au_print/print_hourmeter_exception.jsp
**Line(s):** Header section

## Checklist Items With No Issues
- **1. Secrets and Credentials:** No hardcoded credentials found.
- **2. Authentication and Authorization:** Session-based access controls (access_level, access_cust, access_site, access_dept) applied in all 10 files. Session expiry check via `../sess/Expire.jsp` or `report_export_header.jsp`.
- **3. Injection — SQL Injection:** No direct JDBC or string-concatenated SQL in these JSPs.
- **3. Injection — Command Injection:** No Runtime.exec() or ProcessBuilder.
- **3. Injection — SSRF:** No user-controlled HTTP client calls.
- **7. Dependency Vulnerabilities:** No direct dependency declarations in these JSP files.
