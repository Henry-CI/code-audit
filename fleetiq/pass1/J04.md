### J04-F01: DriverBean fields are package-private — direct field access bypasses encapsulation and serializes without customer scoping
**Description:** In `DriverBean`, all six fields (`id`, `user_cd`, `weigand`, `veh_type`, `vehAccessLvl`, `supervisorAccess`) are declared without any access modifier, making them package-private. Any class in the same package (`com.torrent.surat.fms6.bean`) can read or write these fields directly, bypassing the getter/setter methods entirely. The class implements `Serializable` without a `serialVersionUID`, which can cause silent deserialization mismatches. Additionally, this bean carries no customer/tenant identifier (`customer_id`, `custCd`, `comp_id`), meaning it cannot encode which customer a driver record belongs to — a significant omission in a multi-tenant system.
**Severity:** MEDIUM
**Checklist item:** 2. Authentication and Authorization — Authorization and multi-tenancy; 5. Data Exposure and Privacy
**Evidence:**
```java
// DriverBean.java lines 6–11
String id = "";
String user_cd = "";

```
No `private` modifier. No `customer_id`/`custCd` field anywhere in this class.

### J04-F02: Wiegand card number stored in serializable bean with no access control or encryption
**Description:** `DriverBean` carries a `weigand` field (line 8) representing a Wiegand card number. Wiegand codes are physical access control credentials — they control which doors and vehicles a driver can access. Storing this value as a plain `String` in a serializable Java bean with package-private visibility means: (1) it is transmitted wherever the bean is serialized (HTTP session, cache, any persistence layer); (2) it is accessible to any class in the package without calling a getter; (3) there is no indication it is encrypted or hashed at rest. If rendered to JSP output without escaping, it is also an XSS risk. The access control system that uses `weigand` codes depends on this value remaining confidential — exposure allows physical access control bypass.
**Severity:** HIGH
**Checklist item:** 5. Data Exposure and Privacy — Driver data; 1. Secrets and Credentials
**Evidence:**
```java
// DriverBean.java line 8
String weigand = "";
```
```java

    return weigand;
}

    this.weigand = weigand;
}
```
**File:** `WEB-INF/src/com/torrent/surat/fms6/bean/DriverBean.java`

### J04-F05: DriverImportBean.denyOnExp defaults to false — licence expiry enforcement is opt-in, not opt-out
**Description:** The `denyOnExp` boolean field in `DriverImportBean` (line 31) controls whether a driver with an expired licence is denied vehicle access. Its default is `false`, meaning access is permitted by default when a licence has expired unless `denyOnExp` is explicitly set to `true`. In an import workflow where bean population depends on external file content or HTTP parameter binding, if the `denyOnExp` field is omitted from an import record, the driver will be granted access despite an expired licence. This is a safety and regulatory risk for a fleet management system — drivers with expired licences operating vehicles is a legal liability. The field should default to `true` (deny on expiry) or be mandatory.
**Severity:** MEDIUM

### J04-F08: DriverBean missing serialVersionUID — deserialization instability
**Description:** `DriverBean` implements `Serializable` (line 5) but does not declare a `serialVersionUID` field. Java will auto-generate one based on class structure at compile time. Any change to the class (adding a field, changing a method signature) will change the auto-generated UID, causing `InvalidClassException` when deserializing previously serialized instances. In a session-based web application, this can result in session data being silently dropped or corrupted after a deployment, potentially losing authentication state. This is not a direct security vulnerability but is an operational risk that could mask session management failures.
**Severity:** INFO
**Checklist item:** 4. Session and CSRF — Session management
**Evidence:**
```java
// DriverBean.java line 5 — implements Serializable, no serialVersionUID declared
public class DriverBean implements Serializable{
    String id = "";
    ...
```
Compare `DriverImportBean.java` line 7 which correctly declares:
```java
private static final long serialVersionUID = -6234180688512824118L;
```
**File:** `WEB-INF/src/com/torrent/surat/fms6/bean/DriverBean.java`
