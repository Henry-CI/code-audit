### C01-F08: dealer_url Parameter Passed Unsanitised to Login Page via URL Rewrite
**Description:** The single active rewrite rule in `urlrewrite.xml` redirects requests matching `^/([a-z|_|-]+)$` to `/pages/login.jsp?dealer_url=$1`, where `$1` is the first path segment captured from the request URL. The captured value is appended directly to the redirect URL as the `dealer_url` query parameter. If `login.jsp` reads `request.getParameter("dealer_url")` and uses the value — for example, to customise branding, pre-populate a form field, set a cookie, or perform a redirect after login — this is an open redirect or reflected XSS vector depending on how the JSP processes it. The regex restricts the captured group to `[a-z|_|-]+`, which prevents most XSS payloads via this path, but does not prevent open redirect abuse (the parameter can name any legitimate route or dealer name). The pipe character `|` inside the character class is being treated as a literal pipe, not alternation, which is likely a regex authoring error.
**Severity:** MEDIUM
**Checklist item:** Section 3 (Injection Vulnerabilities) — XSS; user-controlled values passed to JSP output; Section 2 — authentication redirection
**Evidence:**
```xml
<!-- urlrewrite.xml, lines 13-20 -->
<rule>
    <note>
        The rule means that requests to /test/status/ will be redirected to /rewrite-status
        the url will be rewritten.
    </note>
    <from>^/([a-z|_|-]+)$</from>
    <to type="redirect">%{context-path}/pages/login.jsp?dealer_url=$1</to>
</rule>
```
**File:** `WEB-INF/urlrewrite.xml`
**Line(s):** 13-20
