## Findings

### P09-F01: Missing Access Level Control — email_driver_league.jsp
**Description:** In `email_driver_league.jsp`, the session-based access level control block is absent. The comment `//end of access level control` appears at line 45 but no `access_level`, `access_cust`, `access_site`, or `access_dept` values from the session are ever set on the filter bean. This means any authenticated user (regardless of their customer assignment) can trigger the email of driver league data for any customer_cd they provide as a request parameter.
**Severity:** CRITICAL
**Checklist item:** 2. Authentication and Authorization — multi-tenancy
**Evidence:**
```java
// Lines 44-57 — no access control applied:
//end of access level control

filter.setSet_op_code(op_code);
filter.setSet_div_cd("1");
filter.setSet_cust_cd(user_cd);  // user_cd comes from request parameter, not session
filter.setSet_loc_cd(loc_cd);
filter.setSet_dept_cd(dept_cd);
// filter.setAccess_level(...) NEVER CALLED
// filter.setAccess_cust(...) NEVER CALLED
```
**File:** dyn_report/email_driver_league.jsp
**Line(s):** 44–57

### P09-F02: Unescaped Driver/Department Data — XSS in Email Report
**Description:** In `email_driver_league.jsp`, driver names, departments, and truck types from the database are rendered unescaped in the email HTML body.
**Severity:** MEDIUM
**Checklist item:** 3. Injection Vulnerabilities — XSS
**Evidence:**
```jsp
<td align=left><%=bean.getDriverName()%></td>
<td align=left><%=bean.getDepartment()%></td>
<td align=left><%=bean.getTruckType()%></td>
```
**File:** dyn_report/email_driver_league.jsp
**Line(s):** 107–109

### P09-F03: Missing CSRF Protection
**Description:** All email and Excel export endpoints in this partition accept form submissions without CSRF token protection. An attacker can trick authenticated users into triggering email sends or data exports.
**Severity:** HIGH
**Checklist item:** 4. Session and CSRF
**Evidence:** No CSRF token fields present in any file in this partition.
**File:** All files in this partition
**Line(s):** N/A

### P09-F04: CSS File Included as JSP Include (Incorrect Inclusion)
**Description:** `email_driver_league.jsp` includes `<%@ include file="../css/alternate_rows.css"%>`. Including a CSS file as a static JSP include will output raw CSS into the HTML, which may produce malformed HTML. This is a code quality issue that could cause rendering problems but not a direct security vulnerability.
**Severity:** INFO
**Checklist item:** N/A — code quality
**Evidence:**
```jsp
<%@ include file="../css/alternate_rows.css"%>
```
**File:** dyn_report/email_driver_league.jsp
**Line(s):** 3
