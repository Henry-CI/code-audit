### P18-F01: Unescaped session attribute emitted into JavaScript string (XSS)
**Description:** In `header.jsp`, the session attributes `date_format` (lines 333–334) are emitted directly into JavaScript string literals using `<%= %>` without any JavaScript escaping. If a `date_format` value contains a single quote, double quote, backslash, or newline, it breaks out of the string context and allows arbitrary JavaScript execution. Although `date_format` is set server-side at login, any path that writes attacker-influenced data into the session `date_format` attribute would yield stored XSS rendered in the application shell on every page load.
**Severity:** HIGH
**Checklist item:** 3. Injection Vulnerabilities — XSS
**Evidence:**
```jsp
// layout/header.jsp lines 333–334
var dFormat = "<%=session.getAttribute("date_format").toString().toLowerCase().substring(0,SessionVar.date_only_format.length()-2) %>";
var dFormat2 = "<%=session.getAttribute("date_format").toString() %>";
```
**File:** `layout/header.jsp`
**Line(s):** 333–334

### P18-F07: SessionVar fields set as static (class-level) state — multi-tenant isolation failure risk
**Description:** In `header.jsp` lines 39–40, `SessionVar.isSuperAdmin` and `SessionVar.adminAccessLvl` are assigned as what appear to be static fields on the `SessionVar` class (accessed without an instance). In a multi-threaded Tomcat environment, static fields are shared across all concurrent requests. If `SessionVar` uses static fields to hold per-user state (as this code pattern implies), then a high-privilege user's session data can overwrite the values being used by a concurrent low-privilege user's request — potentially granting super-admin access to other users. This is a critical multi-tenancy flaw.
**Severity:** CRITICAL
**Checklist item:** 2. Authentication and Authorization — multi-tenancy; 2. Session management
**Evidence:**
```jsp
// layout/header.jsp lines 39–40
SessionVar.isSuperAdmin = session.getAttribute("admin_access_level") == null ? false:session.getAttribute("admin_access_level").toString().trim().equals("0") ? true:false;

```
**File:** `layout/header.jsp`
**Line(s):** 39–40

**3. Injection — Command injection:** No `Runtime.getRuntime().exec()` or `ProcessBuilder` usage found in any of the three files.
