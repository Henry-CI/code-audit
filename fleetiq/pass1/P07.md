## Findings

### P07-F01: Unescaped Database-Sourced Values — XSS
**Description:** All three print JSP files render database-sourced values (vehicle names, model names, utilization data) unescaped in HTML output.
**Severity:** MEDIUM
**Checklist item:** 3. Injection Vulnerabilities — XSS
**Evidence:**
```jsp
<td><b><%=dsum_driver_nm.get(i) %></b></td>
<td><b><%=dsum_vt_nm.get(i) %></b></td>
```
**File:** au_print/print_util_summary.jsp
**Line(s):** Within table data loop

### P07-F02: Missing CSRF Protection
**Description:** Consistent with application-wide pattern. No CSRF tokens present.
**Severity:** HIGH
**Checklist item:** 4. Session and CSRF
**Evidence:** No CSRF token fields in any of the 3 files.
**File:** All files in this partition
**Line(s):** N/A

- **1. Secrets and Credentials:** No hardcoded credentials found.
- **2. Authentication and Authorization:** Session-based access controls applied in all 3 files.

- **3. Injection — Command Injection:** No Runtime.exec() or ProcessBuilder.

- **7. Dependency Vulnerabilities:** No direct dependency declarations.
