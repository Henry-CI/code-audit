## Findings

### P10-F01: Server-Side Request Forgery (SSRF) — mail_report.jsp
**Description:** In `mail_report.jsp`, the `url` parameter is read directly from the HTTP request and appended to a configured base URL: `url = LindeConfig.emailurl + "/dyn_report/" + url`. This URL is then fetched server-side via `GetHtml.getHTML1(url)`. An attacker can manipulate the `url` parameter to cause the server to fetch arbitrary paths within the email URL domain (path-based SSRF), potentially accessing internal report endpoints or metadata.
**Severity:** HIGH
**Checklist item:** 3. Injection Vulnerabilities — SSRF
**Evidence:**
```java
// Lines 19-22
String url = request.getParameter("url")==null?"":request.getParameter("url");
url = LindeConfig.emailurl+"/dyn_report/"+url;
url = url.replaceAll(" ","+");
// ...
data+=obj.getHTML1(url);  // Server-side HTTP fetch with user-controlled path
```
**File:** dyn_report/mail_report.jsp
**Line(s):** 19–22, 57

### P10-F02: Reflected XSS — mail_id Unescaped in mail_report.jsp
**Description:** `mail_id` is read from the request parameter and rendered unescaped in the HTML response.
**Severity:** HIGH
**Checklist item:** 3. Injection Vulnerabilities — XSS
**Evidence:**
```java
String mail_id=request.getParameter("mail_id");  // line 26
// ...
<font color="white" face='verdana' size=2><b><%=mail_id%></b></font>  // line 77
```
**File:** dyn_report/mail_report.jsp
**Line(s):** 26, 77

### P10-F03: XML Injection in AJAX Endpoints
**Description:** In `get_dept.jsp` and `get_site.jsp`, database values (department/site codes and names) are concatenated directly into XML response strings without XML encoding. If any database value contains `<`, `>`, `&`, or `"`, the XML response will be malformed or inject content.
**Severity:** MEDIUM
**Checklist item:** 3. Injection Vulnerabilities — XSS / XML injection
**Evidence:**
```java
// get_dept.jsp lines 44-45
resp=resp+"<rec>"+"<code>"+tmp+"</code><name>"+tmp1+"</name></rec>";
// tmp and tmp1 are raw database strings, not XML-escaped
```
**File:** dyn_report/get_dept.jsp, dyn_report/get_site.jsp
**Line(s):** 44–45

### P10-F04: Missing CSRF Protection

**Severity:** HIGH
**Checklist item:** 4. Session and CSRF
