## Findings

### P25-F01: Missing Session/Authentication Check on AJAX Endpoints
**Description:** Both `get_assigned_vehicle.jsp` and `get_codes.jsp` are AJAX response endpoints that omit the `<%@ include file="../sess/Expire.jsp" %>` directive present on all other JSP files reviewed. Every other JSP in this batch begins with this include to enforce session expiry. These two endpoints return fleet data (vehicle codes, hire numbers, override codes) without any visible authentication gate.
**Severity:** HIGH
**Checklist item:** Section 2 — Authentication and Authorization
**Evidence:**
```
get_assigned_vehicle.jsp line 1-5: No sess/Expire.jsp include. First lines are:
  <%@ page language="java" contentType="text/html; charset=ISO-8859-1" ...%>

```

**Line(s):** 1–5 in each file

### P25-F02: IDOR — Vehicle Code Accepted from Request Parameter Without Ownership Validation
**Description:** Multiple JSPs accept vehicle-related identifiers (`veh_cd`, `gmtp_id`) directly from request parameters and pass them to the data-access bean with no server-side check that the requested vehicle belongs to the authenticated user's customer. An authenticated user from Customer A could supply the `veh_cd` of a vehicle belonging to Customer B and retrieve or modify that vehicle's data (service settings, production status, CANBus rules). This is a classic Insecure Direct Object Reference vulnerability in a multi-tenant application.
**Severity:** HIGH
**Checklist item:** Section 2 — Authorization and multi-tenancy / IDOR
**Evidence:**
