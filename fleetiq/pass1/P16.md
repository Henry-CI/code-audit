### P16-F01: Unescaped database-sourced values rendered into HTML — Stored XSS
**Description:** Every JSP in this set renders data from the `filter` object (customer names, location names, department names, model names, report names, and their codes) directly into HTML output using raw `<%= ... %>` expressions. There is no use of JSTL `<c:out>`, `${fn:escapeXml()}`, or any other escaping mechanism. These values originate from the database (customer/location/department records), meaning that if any of these database fields contain a `<script>` tag or other HTML payload — whether from a compromised record, a data import, or an admin who stored malicious content — the payload will execute in every user's browser on every page that includes these navigation fragments. All four files are affected. This is stored XSS.
**Severity:** HIGH
**Checklist item:** Section 3 — XSS (Cross-Site Scripting)
**Evidence:**

`includes/report_nevigation_keyword.jsp`, line 98:
```jsp
<option value="<%=Vuser_cd.get(i) %>"><%=Vuser_nm.get(i) %></option>
```


### P16-F07: `dealer_id` session attribute parsed without null-safe integer handling — potential for integer coercion issues
**Description:** In both `report_nevigation_keyword.jsp` (line 39) and `report_nevigation_todate.jsp` (line 48), the `dealer_id` session attribute is retrieved and parsed using `Integer.parseInt(session.getAttribute("dealer_id").toString())` with a ternary null guard. While the null case is handled (defaulting to `0`), the pattern `session.getAttribute("dealer_id").toString()` will throw a `NullPointerException` if the attribute is present but set to `null` explicitly (distinct from absent). More importantly, if `dealer_id` is tampered with in the session or contains a non-integer value (e.g., through session poisoning), `Integer.parseInt` will throw `NumberFormatException`, which may produce an unhandled exception and expose a stack trace. This is an INFO-level robustness finding with a minor data exposure risk if error pages are not configured correctly.
**Severity:** LOW
**Checklist item:** Section 5 — Data Exposure and Privacy (error responses / stack trace exposure)
**Evidence:**

`includes/report_nevigation_keyword.jsp`, line 39:
```java
int dealerId = session.getAttribute("dealer_id") == null ? 0:Integer.parseInt(session.getAttribute("dealer_id").toString());
```

`includes/report_nevigation_todate.jsp`, line 48:
```java
int dealerId = session.getAttribute("dealer_id") == null ? 0:Integer.parseInt(session.getAttribute("dealer_id").toString());
```
**Files:** `includes/report_nevigation_keyword.jsp`, `includes/report_nevigation_todate.jsp`
**Lines:** keyword.jsp: 39; todate.jsp: 48.
