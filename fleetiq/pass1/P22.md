### P22-F01: Reflected XSS — `message` parameter rendered unescaped in HTML across all files
**Description:** Every JSP in this batch reads a `message` request parameter and renders it directly into the HTML response using `<%= message %>` with no escaping. An attacker can craft a URL such as `frm_customer_rel.jsp?message=<script>alert(1)</script>` and deliver it to a victim user. Since these are authenticated pages (behind `Expire.jsp`), the attack vector is a phishing link or open-redirect chained scenario targeting authenticated sessions.
**Severity:** HIGH
**Checklist item:** 3. Injection Vulnerabilities — XSS

### P22-F02: Reflected XSS — URL parameters emitted unescaped into JavaScript `onload` / `onclick` attribute contexts
**Description:** Multiple JSPs emit user-controlled request parameters directly into JavaScript function call arguments in HTML event handler attributes (`onload`, `onclick`). A parameter value containing a single quote such as `user_cd='+alert(1)+'` terminates the JavaScript string and injects arbitrary code. ## Checklist Items With No Issues

The following checklist items were reviewed and had no findings within the ten files in scope for this agent:

- **1. Secrets and Credentials:** No hardcoded database passwords, API keys, SMTP credentials, or connection strings were found in any of the ten JSP files. No `.env` file references or embedded credentials in comments were observed.

- **3. Injection — File Inclusion:** All `<%@ include file="..." %>` and `<jsp:include>` directives use static, hardcoded paths (`../sess/Expire.jsp`, `../menu/menu.jsp`). No user-controlled file path inclusion was observed.

- **3. Injection — Command Injection:** No calls to `Runtime.getRuntime().exec()` or `ProcessBuilder` were found in any of the ten files.

SQL injection risk in those beans is out of scope for this agent's file set but should be assessed when those Java source files are reviewed.

- **6. Transport Security:** No transport configuration (TLS settings, connector configuration) is present in JSP files. Transport security must be assessed in `server.xml` and `context.xml`, which are not in this agent's file set.

- **7. Dependency Vulnerabilities:** No dependency declarations are present in JSP files. Library versions must be assessed from `pom.xml`, `build.xml`, or committed JARs by the appropriate agent.
