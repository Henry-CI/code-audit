## Findings

### P62-F01: Cross-Site Scripting (XSS) - Unescaped Database Output in Question Display
**Description:** The view.jsp file outputs question text and expected answers directly from the database without HTML escaping. When displaying checklist questions, the question text (`questionBean.getQuestion()`) and expected answer (`questionBean.getExp_ans()`) are output using `<%= %>` without any encoding. Since these values are user-supplied data stored in the database (editable via update_question.jsp), a malicious user could inject JavaScript that would execute when other users view the vehicle details.

**Severity:** HIGH

**Checklist item:** 3. Injection Vulnerabilities - XSS (Cross-Site Scripting)

**Evidence:**
```jsp
// view.jsp
Line 277: <td class="acenter"><%=questionBean.getQuestion()%></td>
Line 279: <td class="acenter" width="10%"><%=questionBean.getExp_ans()%></td>
```

**File:** pages/admin/vehicles/view.jsp

**Line(s):** 277, 279

### P62-F03: Cross-Site Scripting (XSS) - Unescaped Database Output in VOR Note Field
**Description:** The vor-settings.jsp file outputs the VOR note field directly from the database without HTML escaping. The note value is retrieved from the database and rendered in a textarea using `<%= %>`. While textarea content is generally safer than direct HTML output, the note value is also potentially used elsewhere in the application. If this note is ever displayed outside a textarea context without escaping, it would create an XSS vulnerability.

**Severity:** MEDIUM

**Checklist item:** 3. Injection Vulnerabilities - XSS (Cross-Site Scripting)

**Evidence:**
