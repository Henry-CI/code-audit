**Description:** The `checkExpiry` method builds a `debugCond` string fragment by directly concatenating the `userCd` and `weigand` method parameters (lines 42–43). ### J74-F11: Weak PRNG used for password/token generation
**Description:** `DataUtil.getRandomString(int length)` (line 440) uses `new Random(System.currentTimeMillis())` to seed a `java.util.Random` instance. `java.util.Random` is a linear-congruential generator and is not cryptographically secure. Seeding with `System.currentTimeMillis()` makes it predictable: an attacker who knows the approximate time of token generation can brute-force the seed space. This method is intended for generating random strings (the `charset` field includes digits, symbols, and letters, suggesting use for passwords or tokens). `java.security.SecureRandom` should be used instead.
**Severity:** HIGH
**Checklist item:** 2. Authentication and Authorization — session management / credential generation
**Evidence:**
```java
// Lines 440-448
public static String getRandomString(int length) {
    Random rand = new Random(System.currentTimeMillis());  // weak PRNG, time-seeded
    StringBuffer sb = new StringBuffer();
    for (int i = 0; i <= length; i++) {
        int pos = rand.nextInt(charset.length());
        sb.append(charset.charAt(pos));
    }
    return sb.toString();
}
```
