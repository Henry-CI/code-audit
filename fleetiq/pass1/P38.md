### P38-F01: Unescaped Database Values Output in HTML â€” Stored XSS Risk Across All Report Pages
**Description:** Every report JSP in this set outputs data retrieved from the database directly via `<%= ... %>` expressions without HTML escaping. Vehicle names, driver names, battery fleet numbers, broadcast message text, model names, and all other data fields are rendered raw into HTML. If any of these database fields contain malicious content (injected via another input vector, or entered by a privileged user), it will execute as JavaScript in the browser of any user viewing the report. The broadcast message text (`broadcastmsgBean.getText()`) is particularly high-risk as it represents user-authored message content.
**Severity:** HIGH
**Checklist item:** Section 3 â€” XSS (Cross-Site Scripting)
**Evidence:**
```
// battery_change_report.jsp lines 112-123
<td><%=batteryBean.getUnit_nm() %></td>
<td><%=batteryBean.getBef_dri_nm() %></td>

<td><%=broadcastmsgBean.getText() %></td>   // user-authored message body
```
