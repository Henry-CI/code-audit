### J84-F01: Broken SQL Injection Mitigation — `escapeSingleQuotes` Does Not Prevent SQL Injection

**Description:** The `escapeSingleQuotes.replaceSingleQuotes()` method is designed to sanitise user input before SQL string concatenation. It attempts to escape single quotes by doubling them (`'` becomes `''`). This is a classic but fundamentally flawed approach to SQL injection prevention. It does not prevent SQL injection in databases that accept other quoting styles or escape sequences. It also provides zero protection against injection via non-string SQL parameters (integers, booleans). The existence of this class strongly implies that calling code uses string concatenation for SQL queries rather than parameterised queries — the correct fix is to eliminate string concatenation entirely with PreparedStatement. The class itself is evidence of an anti-pattern in use.

**Severity:** HIGH

**Checklist item:** Section 3 — Injection Vulnerabilities (SQL Injection)

**Evidence:**
```java
// Lines 4–17
public String replaceSingleQuotes(String sInput){
      StringBuffer sbOutput = new StringBuffer();
      for(int i = 0; sInput!= null && i < sInput.length(); i++)
      {
          sbOutput.append(sInput.charAt(i));
          // If the character is a single quote
          // escape it.
          if(sInput.charAt(i) == '\''){
              sbOutput.append('\'');
          }

      }
      return sbOutput.toString();
}
```
