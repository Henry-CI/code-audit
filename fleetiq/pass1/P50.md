### P50-F01: SSRF via User-Controlled URL Path in mail_report.jsp
**Description:** In `mail_report.jsp`, the `url` request parameter is read directly and appended to `LindeConfig.emailurl` (a configured base URL, e.g. `http://fms.fleetiq360.com`) to form a full URL. This URL is then fetched server-side using `GetHtml.getHTML1()` which calls `HttpURLConnection`. An authenticated attacker can supply a crafted `url` parameter (e.g., containing `../`, URL-encoded path traversal, or query string overrides) to redirect the server-side HTTP request to arbitrary paths on the same host or, depending on path handling, bypass the base URL prefix. The constructed URL is formed without any validation or whitelist check on the path component. The response body (arbitrary HTML content) is then sent as the email body to the `mail_id` recipient.
**Severity:** HIGH
**Checklist item:** Section 3 ‚Äî SSRF (Server-Side Request Forgery)
**Evidence:**
