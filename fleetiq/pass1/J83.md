### J83-F01: Pervasive SQL Injection via String Concatenation — UtilBean.java
**Description:** Every SQL query in `UtilBean.java` is constructed by string concatenation rather than parameterised `PreparedStatement`. This affects multiple methods and multiple data types: `SessionVar.dealerId` (an application-wide session variable), `userCd` (passed as a method parameter), `webUserCD` (passed as a method parameter), `vehicleID` (passed as a method parameter), and `vehCd` (passed as a method parameter). Any path through which an attacker can control these values — directly or via a second-order injection — allows arbitrary SQL execution. The `getTimezone2` and `getLocalTime` methods accept parameters that flow directly into queries without any sanitisation.
**Severity:** CRITICAL
**Checklist item:** 3. Injection Vulnerabilities — SQL injection
**Evidence:**
