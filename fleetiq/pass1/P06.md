## Findings

### P06-F01: Unescaped Database Attribute Injection in bgc Variable (HTML Injection)
**Description:** In `print_impact_meter_report.jsp` and `print_impact_report.jsp`, a `bgc` string variable is constructed from a database-sourced severity string and inserted raw into an HTML element's attribute position: `<td ... <%=bgc %>>`. While the current database values appear safe ("Red Level", "Amber Level", "Blue Level"), any modification to the database to store a crafted string could allow HTML attribute injection.
**Severity:** MEDIUM
**Checklist item:** 3. Injection Vulnerabilities — XSS
**Evidence:**
```jsp
String bgc = "";
if(tmp.equalsIgnoreCase("Red Level")) { bgc = "bgcolor='red'"; }
// ...
<td width = 20% <%=bgc %>><%=tmp %>
```
**File:** au_print/print_impact_meter_report.jsp
**Line(s):** 144–160

### P06-F02: Unescaped Lock Reason Text — Stored XSS Risk
**Description:** In `print_unit_unlock.jsp`, the unit lock reason (which is entered by operators or drivers in the field) is rendered unescaped. This constitutes a stored XSS risk if drivers or administrators can enter HTML special characters in the lock reason field.
**Severity:** MEDIUM
**Checklist item:** 3. Injection Vulnerabilities — XSS (stored)
**Evidence:**
```jsp
<td><%=vunit_loreason.get(i) %></td>
```
**File:** au_print/print_unit_unlock.jsp
**Line(s):** Within data table loop

### P06-F03: Unescaped Pre-Op Question Text — Stored XSS Risk
**Description:** In `print_preop_chk.jsp` and `print_preop_chk_fail.jsp`, pre-operation question text is rendered unescaped. Questions are entered by administrators and could contain HTML.
