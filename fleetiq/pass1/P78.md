### P78-F01: Cross-Site Scripting (XSS) via Unescaped JSP Expressions in HTML Context
**Description:** Multiple JSP files output user-controlled or database-derived values directly into HTML without proper escaping. An attacker who can control customer names, location names, department names, vehicle names, driver names, or other data stored in the database could inject malicious HTML/JavaScript that executes in other users' browsers. This violates the principle of defense-in-depth even if the data is sanitized on input, as output encoding is the primary defense against XSS.

### P78-F06: Client-Side URL Construction for AJAX with User-Controllable Parameters
**Description:** AJAX requests to get_driverblacklist.jsp construct the URL using values from hidden form fields (cust_cd, loc_cd, dept_cd, scr). While these values are initially populated from server-side data, they can be manipulated by the user via browser developer tools before the AJAX call executes. If the backend endpoint get_driverblacklist.jsp does not properly validate and enforce authorization based on session attributes (rather than trusting the URL parameters), this could lead to unauthorized data access across customer boundaries.

**Severity:** MEDIUM

**Checklist item:** Section 2 - Authentication and Authorization (multi-tenancy)
