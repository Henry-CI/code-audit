### P34-F01: Pervasive Missing CSRF Protection Across All State-Changing Forms
**Description:** None of the state-changing forms in this file set include a CSRF token — neither a hidden field with a server-generated token nor any equivalent mechanism. Any authenticated user can be tricked into submitting a forged cross-site request to create, edit, or delete customers, departments, sites, access groups, or site-hour settings. Because these are admin-level functions managing the customer/tenant master data in a multi-tenant system, successful CSRF against an admin could allow an attacker to create rogue customers, modify existing customer details, manipulate site authorisation, or delete departments.

### P34-F02: Reflected XSS — Unescaped Request Parameters Written Into HTML and JavaScript
**Description:** Multiple JSP files reflect request parameters directly into HTML form field values and JavaScript variable assignments using raw `<%= ... %>` without HTML or JavaScript encoding. An attacker who can control the URL parameters (e.g., via a crafted link or CSRF chain) can inject arbitrary HTML or JavaScript.
