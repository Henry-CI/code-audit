### P86-F01: Missing Session Guard on auto_change_pass.jsp
**Description:** `security/auto_change_pass.jsp` does not include `../sess/Expire.jsp`, which is the session expiry/authentication guard used by all other files in this directory. All nine other files in this set include it. Without this guard, the page may be accessible to unauthenticated users or to users with expired sessions. While session attributes are read (e.g. `user_cd`), the absence of the session guard means no server-side enforcement of authentication state is applied at the page level.

**Checklist item:** Section 2 — Authentication and Authorization

### P86-F03: Reflected XSS — Unescaped `message` Parameter Output Across All Files
**Description:** Every file in this set outputs `request.getParameter("message")` directly using `<%= message %>` with no HTML escaping. An attacker who can craft a link to any of these pages with a malicious `message` parameter value (e.g. `message=<script>alert(1)</script>`) will have that script executed in the victim's browser. The `message` parameter is used as a feedback mechanism (e.g. after a failed save operation, the servlet redirects back with a `message` parameter). This is a classic reflected XSS vector.
**Severity:** HIGH
**Checklist item:** Section 3 — XSS (Cross-Site Scripting)
**Evidence (representative — same pattern in all 10 files):**
