### J87-F01: Pervasive SQL Injection — DriverImportDAO Driver Name and Licence Search Methods
**Description:** The `checkDriverByNm`, `checkDriverByLic`, `getDriverByNm`, and `getDriverByFullNm` methods in `DriverImportDAO` build SQL queries by directly concatenating the `fname`, `lname`, and `licence` parameters into the query string. These parameters originate from import file content (CSV values) but may also be influenced by user input at the import stage. The `ilike` operator and string embedding make these fully exploitable for SQL injection — an attacker controlling the first name or licence field in an import file can escape the string literal and inject arbitrary SQL.
**Severity:** HIGH
**Checklist item:** 3. Injection Vulnerabilities — SQL injection
**Evidence:**
```java
// DriverImportDAO.java line 29-31
String sql = "select id from driver where trim(both ' ' from first_name) ilike trim ( both ' ' from '"
        + fname + "')" + "  and trim(both ' ' from last_name) ilike trim ( both ' ' from '" + lname + "')"
        + "  and comp_id = " + compId;

// line 83
String sql = "select id from driver where trim(both ' ' from licno) ilike trim(both ' ' from '" + licence + "')" + "  and comp_id = " + compId;

// line 136-138
String sql = "select id,first_name,last_name,active,comp_id from driver where trim(both ' ' from first_name) ilike trim(both ' ' from '"
        + fname + "')  " + " and trim(both ' ' from last_name) ilike trim(both ' ' from '"
        + lname + "')" + " and comp_id = " + compId;

// line 192-193
String sql = "select id,first_name,last_name,active,comp_id,licno from driver where first_name||' '||last_name ilike '"
        + fname + "' and comp_id = " + compId;
```
