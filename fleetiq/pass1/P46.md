## Findings

### P46-F01: Unvalidated email address reflected in JSON response (JSON injection / header injection vector)
**Description:** In all 10 JSP files, the `mail_id` parameter is read directly from the HTTP request with no validation, sanitisation, or encoding, and is then concatenated verbatim into the JSON response string that is written to the client. An attacker who can control the `mail_id` parameter can break out of the JSON string and inject arbitrary JSON keys/values, or inject content that a consuming client may misinterpret. In addition, `mail_id` is passed directly to `MailExcelReports.sendExcelReport()` as the recipient address with no format validation, which could enable email header injection if the mail library does not sanitise it (CRLF injection into `To:` header).
**Severity:** HIGH
**Checklist item:** Section 3 — Injection Vulnerabilities (XSS / output injection)
**Evidence:**
```java
// xlsx_driver_license_report_mail.jsp lines 59, 72
String mail_id=request.getParameter("mail_id");
...
message = "{ \"status\":\"success_close\", \"message\":\"Mail has been sent to " + mail_id + ".\" }";
```
