### J66-F08: Race Condition — SingleThreadModel on Shared Mutable Instance Fields

**Description:** `Frm_saveuser` implements the deprecated `javax.servlet.SingleThreadModel` interface (line 39). This means the container serializes requests through a single instance (or pools instances), but the intent is to protect shared mutable instance fields (`dbcon`, `stmt`, `rset`, `queryString`, `message`, `url`, etc.). However, `SingleThreadModel` is deprecated since Servlet 2.4 and its behavior is unreliable — some containers pool multiple instances, causing the same race conditions the interface is supposed to prevent. The real fix is to use local variables for all per-request state, which is not done here. Under load, two concurrent requests can share a `stmt` or `rset`, leading to data from one user's request leaking into another user's response, or to corrupted query strings.

**Severity:** HIGH

**Checklist item:** 2. Authentication and Authorization; 5. Data Exposure and Privacy

**Evidence (lines 22, 39, 42–55):**
