### P58-F01: Cross-Site Scripting (XSS) — Unescaped Expression Outputs Throughout All JSP Files
**Description:** All reviewed JSP files contain numerous instances of `<%= ... %>` expression outputs that render user-controlled or database-sourced data directly into HTML without proper escaping. This allows malicious content stored in the database (via vehicle names, driver names, customer names, questions, etc.) to execute arbitrary JavaScript in users' browsers. An attacker who can inject malicious content into any of these fields can perform session hijacking, credential theft, or defacement.

**Severity:** HIGH

**Checklist item:** 3. Injection Vulnerabilities — XSS (Cross-Site Scripting)

**Evidence:**

add-general.jsp:
```jsp
80→  <option value="<%=Fms_Usr_Mst_Cd.get(i)%>"<%=( customer.equals( Fms_Usr_Mst_Cd.get(i).toString() ) ? " selected=\"true\"" : "" )%>><%=Fms_Usr_Mst_Name.get(i)%></option>

An attacker could manipulate these parameters to potentially access or modify data belonging to other customers/tenants.

## Summary

This security audit of 10 JSP files in the FleetIQ vehicle management section identified **10 findings** ranging from HIGH to INFO severity:

- **HIGH severity (3 findings):** XSS via unescaped outputs (F01), XSS via JavaScript interpolation (F02), Missing CSRF protection (F03)
- **MEDIUM severity (5 findings):** Parameter tampering/authorization (F04), Sensitive operations via JavaScript (F06), Permission array validation (F07), Request parameters in attributes (F08), Session attribute handling (F09)
- **LOW severity (1 finding):** Wildcard imports (F05)
- **INFO severity (1 finding):** Site-specific conditional logic (F10)

The most critical issues are the widespread lack of output escaping (XSS) and the complete absence of CSRF tokens on state-changing forms. These should be prioritized for remediation.

Several checklist items could not be fully assessed from JSP review alone and require backend Java class review (SQL injection), configuration file review (transport security, error handling), and dependency inventory (vulnerability scanning).
