## Findings

### P03-F01: Unvalidated Numeric Parameter — sev (Impact Severity)
**Description:** In `xlsx_driver_impact_report.jsp` and `xlsx_impact_meter_report.jsp`, the `sev` parameter is read directly from the request and passed to `filter.setSet_imp_sev(sev)` without validation that it is a numeric value. If the downstream DAO uses this in a SQL query without parameterization, this could be an injection vector. Even if parameterized, lack of input validation is a code quality issue.
**Severity:** MEDIUM
**Checklist item:** 3. Injection Vulnerabilities — SQL injection
**Evidence:**
```java
// xlsx_driver_impact_report.jsp line 26-27
String sev = request.getParameter("sev") == null ? "10" : request.getParameter("sev");
// ...
filter.setSet_imp_sev(sev);  // passed to DAO without numeric validation
```
**File:** au_excel/xlsx_driver_impact_report.jsp, au_excel/xlsx_impact_meter_report.jsp
**Line(s):** 26–27, 62 (driver_impact); 28–29, 63 (impact_meter)

### P03-F02: Missing CSRF Protection — Report Download Endpoints
**Description:** All ten JSP files in this partition accept POST requests to generate and stream Excel reports. None contain CSRF tokens. An attacker can trick an authenticated user into triggering report downloads via cross-site form submission.
**Severity:** HIGH
**Checklist item:** 4. Session and CSRF
**Evidence:** No CSRF token fields present in any file. All files are invoked via POST from parent report pages.
**File:** au_excel/xlsx_driver_impact_report.jsp (and all others in partition)
**Line(s):** N/A — absence throughout

### P03-F03: Stack Trace Exposure via e.printStackTrace()
**Description:** All files catch `Exception` during file I/O and call `e.printStackTrace()`. This logs internal stack trace information that could aid an attacker in mapping the application's internal structure.
**Severity:** LOW
**Checklist item:** 5. Data Exposure and Privacy — Error responses
**Evidence:**
```java
} catch (Exception e) // file IO errors
{
    e.printStackTrace();
}
```
**File:** au_excel/xlsx_driver_impact_report.jsp (repeated in all 10 files)
**Line(s):** 107–110 (driver_impact); similar in all files

## Checklist Items With No Issues
- **1. Secrets and Credentials:** No hardcoded credentials or API keys found.
- **2. Authentication and Authorization:** Session-based access controls (access_level, access_cust, access_site, access_dept) are present and applied in all files in this partition. No commented-out access control blocks observed (unlike xlsx_curr_unit_report.jsp in P02).
- **3. Injection — XSS:** No HTML output rendered in these files; they stream binary Excel content directly to the response.
- **3. Injection — Command Injection:** No Runtime.exec() or ProcessBuilder usage.
- **3. Injection — SSRF:** No HTTP client calls with user-controlled URLs.
- **6. Transport Security:** No transport configuration in these JSP files.
- **7. Dependency Vulnerabilities:** No direct dependency declarations in these JSP files.
