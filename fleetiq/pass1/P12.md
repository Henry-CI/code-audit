### P12-F05: Unescaped database values written to CSV response — CSV Injection
**Description:** In all three exporters, data values retrieved from the database are written directly to the HTTP response via `o.print(...)` without any sanitisation or escaping for CSV format. Specifically:

### P12-F10: Incorrect Content-Type on checklist.jsp and vehicles.jsp CSV exports
**Description:** Both `checklist.jsp` (line 9) and `vehicles.jsp` (line 9) set `Content-Type: text/html;charset=utf-8` while serving CSV data. The correct MIME type for CSV is `text/csv` or `application/octet-stream`. Using `text/html` causes the browser to potentially interpret the response as HTML. If any CSV cell value contains HTML-like content (e.g. `<script>` in a vehicle name or checklist question), the browser may render it as HTML rather than download it, creating a stored XSS vector for users who navigate directly to the export URL.

**Severity:** MEDIUM
**Checklist item:** Section 3 — XSS
**Evidence:**
```
exporter/checklist.jsp line 9:
    response.setContentType("text/html;charset=utf-8");

exporter/vehicles.jsp line 9:
    response.setContentType("text/html;charset=utf-8");
```

### P12-F11: No CSRF protection on export endpoints
**Description:** All three export endpoints are JSP files that respond to HTTP GET requests. They produce side-effect-free CSV downloads, so CSRF is a lower risk than for state-changing endpoints. However, the lack of any CSRF token means an attacker can trick an authenticated user into triggering a download from a third-party page (e.g. an `<img>` or `<iframe>` tag pointing to the export URL). Combined with the IDOR vulnerabilities above (P12-F06, P12-F07), an attacker could craft a URL that exfiltrates another tenant's user data via the victim's authenticated session. No CSRF token or `SameSite` cookie attribute was observed protecting against this.

**Severity:** MEDIUM
**Checklist item:** Section 4 — Session and CSRF
**Evidence:**
```
exporter/checklist.jsp: no CSRF token check present
exporter/users.jsp: no CSRF token check present
exporter/vehicles.jsp: no CSRF token check present
```
**File:** `exporter/checklist.jsp`, `exporter/users.jsp`, `exporter/vehicles.jsp`
**Line(s):** All three files — entire page
