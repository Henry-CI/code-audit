### P19-F01: Reflected XSS via Unescaped Request Parameter `message` in Multiple JSPs
**Description:** The `message` parameter is read directly from the HTTP request and rendered into the page HTML without any escaping. Any HTML or JavaScript injected into this parameter will execute in the victim's browser. The affected files include `copy_site_hour_settings.jsp`, `edit_dept_name.jsp`, `edit_site_address.jsp`, `edit_site_name.jsp`, `existing_alert_lst.jsp`, and `existing_alert_lst_admin.jsp`. In `edit_dept_name.jsp` the message is output with no enclosing tag at all (bare `<%=message %>`), allowing full HTML injection.
**Severity:** HIGH
**Checklist item:** 3. Injection Vulnerabilities — XSS (Cross-Site Scripting)
**Evidence:**
```jsp
// copy_site_hour_settings.jsp line 104
<p style="color: red;font-weight:bold;font-size:16px" align="center"><%=message %></p>

// edit_dept_name.jsp line 54 — no enclosing tag
<%=message %>

// edit_site_address.jsp line 55
<%=message %>

// edit_site_name.jsp line 55
<%=message %>

// existing_alert_lst.jsp line 172
&nbsp; <%=message %>

// existing_alert_lst_admin.jsp line 465
&nbsp; <%=message %>
```
**File:** master/copy_site_hour_settings.jsp, master/edit_dept_name.jsp, master/edit_site_address.jsp, master/edit_site_name.jsp, master/existing_alert_lst.jsp, master/existing_alert_lst_admin.jsp

### P19-F02: Reflected XSS via Unescaped `location_nm` and `state` Request Parameters

### P19-F03: JavaScript Injection via `form_cd` Request Parameter in Event Handlers
**Description:** In `edit_dept_name.jsp`, `edit_site_address.jsp`, and `edit_site_name.jsp`, the `form_cd` request parameter is embedded unescaped directly inside a JavaScript string literal within an `onclick` event handler attribute: `close_form('<%=form_cd %>')`. If `form_cd` contains a single-quote followed by JavaScript, this breaks out of the string literal and allows script injection. This is XSS via JavaScript context injection.
**Severity:** HIGH
**Checklist item:** 3. Injection Vulnerabilities — XSS (Cross-Site Scripting)
**Evidence:**
```jsp
// edit_dept_name.jsp line 58
<input type="button" name="b2" value="Close" onclick="close_form('<%=form_cd %>');">

// edit_site_address.jsp line 59
```

**File:** master/edit_dept_name.jsp, master/edit_site_address.jsp, master/edit_site_name.jsp

### P19-F04: JavaScript Injection via Database Values Embedded in JS Event Handlers (`alert_id`, `veh_cd`, etc.)
**Description:** Multiple JSPs inject database-sourced values directly into JavaScript event handler strings without any escaping. In `existing_alert_lst.jsp` and `existing_alert_lst_admin.jsp`, alert IDs are placed inside `open_edit('...')` and `submit_del('...')` JS calls. In `existing_cust_vehicle_lst.jsp`, vehicle codes, GMTP IDs, and unit flags are injected into `open_edit(...)`, `open_repl(...)`, and `open_canb(...)`. If any of these database values contain a single-quote or other JavaScript metacharacter (e.g., if a data entry was performed with an injection payload), the JavaScript will break and execute attacker-controlled code in every user's browser that views the list.
**Severity:** HIGH
**Checklist item:** 3. Injection Vulnerabilities — XSS (Cross-Site Scripting)
**Evidence:**
