### P24-F01: Reflected XSS via Unescaped `message` Parameter Across All Files
**Description:** Every JSP in this batch renders the `message` request parameter directly to the HTML page using `<%= message %>` with no HTML escaping. An attacker who can craft a link with a malicious `message` parameter value (e.g., `message=<script>...</script>`) will have that script executed in the victim's browser. Since these forms redirect back to themselves on save failure with the message in the URL, this is an exploitable reflected XSS vector.
**Severity:** HIGH
**Checklist item:** Section 3 — XSS (Cross-Site Scripting)
