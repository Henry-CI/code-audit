## Findings

### P04-F01: Missing CSRF Protection — Report Download Endpoints
**Description:** All five files accept form submissions (POST) to generate and stream Excel reports. None contain CSRF token fields. This is consistent with the application-wide absence of CSRF protection.
**Severity:** HIGH
**Checklist item:** 4. Session and CSRF
**Evidence:** No CSRF token fields present in any file in this partition.
**File:** au_excel/xlsx_serv_maintenance_report.jsp (pattern across all 5 files)
**Line(s):** N/A — absence throughout

### P04-F02: Stack Trace Exposure via e.printStackTrace()
**Description:** All five files catch `Exception` during file I/O and call `e.printStackTrace()`.
**Severity:** LOW
**Checklist item:** 5. Data Exposure and Privacy — Error responses
**Evidence:**
```java
} catch (Exception e) // file IO errors
{
    e.printStackTrace();
}
```
**File:** au_excel/xlsx_serv_maintenance_report.jsp (repeated in all 5 files)
**Line(s):** Pattern consistent across all files

### P04-F03: Parameter Name Mismatch — Potential Confusion in serv_maintenance
**Description:** In `xlsx_serv_maintenance_report.jsp`, the customer code is read from the parameter named `cust_cd` but assigned to a variable named `user_cd` (line 25). This naming inconsistency could cause confusion about what data is being filtered and could mask authorization errors if the `cust_cd` parameter is absent while a session `user_cd` is expected.
**Severity:** LOW
**Checklist item:** 2. Authentication and Authorization
**Evidence:**
```java
// Line 25
String user_cd = request.getParameter("cust_cd")==null?"":request.getParameter("cust_cd");
```
**File:** au_excel/xlsx_serv_maintenance_report.jsp
**Line(s):** 25
