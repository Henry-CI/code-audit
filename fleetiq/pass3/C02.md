There is no explanation of the file's purpose, which application/module it belongs to, the locale it represents, naming conventions for property keys, or how to properly add new entries.
**Fix:** Add a comprehensive file header comment block at the beginning of each file explaining:
- Purpose: "Application-wide internationalization (i18n) message bundle for FleetIQ"

- Key naming conventions: Document the prefix patterns (gen., global., nav., report., user., vehicle., customer., admin., dashboard., alert., driver.)
- Usage instructions: How to reference properties in JSP files and Java code
- Maintenance guidelines: How to add new properties and maintain consistency across locale files

4. Standardize spelling: decide on either "Utilization" (US) or "Utilisation" (UK/International) for each locale file
5. Ensure all section headers and comments are identical across files (only property values should differ)

### C02-F11: Missing Documentation for Locale Fallback Behavior
**Severity:** LOW
**Description:** The properties files do not document the locale fallback behavior of Java ResourceBundles. Developers may not understand that Messages_en_US.properties falls back to Messages_en.properties, which falls back to Messages.properties. This can lead to confusion about which file should contain which properties and whether all properties need to be duplicated across files.
**Fix:** Add a comment section in the file header explaining the locale fallback hierarchy:
```
# Locale Fallback Hierarchy:
# 1. Messages.properties - Default/base locale (used when no matching locale is found)
# 2. Messages_en.properties - English locale (inherits from default, overrides specific keys)
# 3. Messages_en_US.properties - US English locale (inherits from English, overrides specific keys)
#
# Note: Only include properties in locale-specific files if they differ from the parent.
# Currently all files contain all properties for consistency, but this could be optimized.
```
**File:** WEB-INF/src/main/resource/Messages.properties, Messages_en.properties, Messages_en_US.properties
**Line(s):** 1-5 (add to file header)
**Method:** N/A
