### P57-F04: Missing documentation for fnChangeSelect() function
**Severity:** MEDIUM
**Description:** The JavaScript function `fnChangeSelect(type, obj)` in veh_access.jsp has no documentation comment. This complex function (133 lines) implements cascading checkbox deselection logic for departments, vehicle types, and vehicles. The function's logic is non-trivial: when a user selects an item of one type (department, vehicle type, or vehicle), it clears selections from the other two types to maintain mutual exclusivity. Without documentation, the purpose and complex interaction logic are difficult to understand.
**Fix:** Add a JSDoc comment block immediately before the function definition (line 126) describing:
- Function purpose: manages mutually exclusive selection behavior among departments, vehicle types, and vehicles
- Parameter @param {string} type - selection type indicator: 'dpt' (department), 'vt' (vehicle type), or 'veh' (vehicle)
- Parameter @param {object|string} obj - either the triggering DOM element or string 'all' for select-all scenarios
- Logic: when one category is selected, clears active state and values from the other two categories
- Dependencies: reads hidden input values dtsize, vtsize, vsize to determine iteration counts
**File:** pages/admin/user/veh_access.jsp
**Line(s):** 126
**Method:** fnChangeSelect(type, obj)
