### J32-F07: Thread-safety concerns with instance variables in Pct_utilisation
**Severity:** HIGH
**Description:** The Pct_utilisation class declares multiple instance variables (cusList, rsList, dateList, vehicles, vehNum, drivers, thresholdLow, thresholdHigh, oneDay, dealer, level, sessionCust) at lines 36-46. Servlets are singleton objects shared across multiple requests, making these instance variables subject to race conditions when accessed by concurrent requests. Unlike Impacts and Licence which properly use ConcurrentHashMap with session IDs, this servlet stores request-scoped data as instance fields, which is a critical thread-safety issue. While not strictly a documentation gap, this represents a significant implementation flaw that should be documented as a known issue until fixed.
**Fix:** Document this architectural issue in the class-level Javadoc and recommend refactoring:
```java
/**
 * Servlet for displaying vehicle utilization percentage statistics.
 *
 * WARNING: This servlet currently uses instance variables for request-scoped data (lines 36-46),
 * which creates thread-safety issues since servlets are singletons. Multiple concurrent requests
 * will interfere with each other's data. This should be refactored to use local variables within
 * methods or session-scoped storage similar to the Impacts and Licence servlets.
 *
 * RECOMMENDED: Refactor to use ConcurrentHashMap<sessionId, data> pattern as seen in Impacts.java
 * and Licence.java for thread-safe session management.
 */
```
