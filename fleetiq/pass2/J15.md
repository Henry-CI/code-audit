### J15-F04: Critical ArrayIndexOutOfBoundsException risk in UnitutilBean.setUtil()
**Severity:** CRITICAL
**Description:** The method setUtil(int i, int util) at line 78 directly accesses the util array without bounds checking: `this.util[i] = util`. This method has zero test coverage and is called from DAO classes (UnitDAO, ImpactDAO). If a caller passes an invalid index (i < 0 or i >= 8), an ArrayIndexOutOfBoundsException will occur at runtime. This is a critical defect that could cause application crashes when processing utilisation data. No validation exists to prevent this error condition.
**Fix:** Add validation to setUtil() method to check bounds: `if (i < 0 || i >= util.length) throw new IllegalArgumentException("Index out of bounds: " + i)`. Then create test cases in UnitutilBeanTest.java covering: (1) valid indices 0-7 with various values, (2) invalid index -1 expecting IllegalArgumentException, (3) invalid index 8 expecting IllegalArgumentException, (4) invalid index Integer.MAX_VALUE, (5) verify values are correctly stored and retrievable via getUtil().
**File:** WEB-INF/src/com/torrent/surat/fms6/bean/UnitutilBean.java
