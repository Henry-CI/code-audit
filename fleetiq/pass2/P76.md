### P76-F14: Untested HTML/JavaScript injection in dynamic content
**Severity:** HIGH
**Description:** Dynamic data from database queries is directly output to HTML without verified escaping (e.g., `<%=mach_nm.get(i) %>`, `<%=driv_nm.get(i) %>`, `<%=get_user %>`). If database contains malicious JavaScript or HTML, it will be executed in user browsers (XSS vulnerability). No tests verify proper HTML escaping or XSS prevention.
**Fix:** Implement XSS prevention test suite: (1) Test that special HTML characters (<, >, &, ", ') are properly escaped in all dynamic output; (2) Test JavaScript payloads (<script>alert('XSS')</script>) in all input fields are neutralized; (3) Test HTML injection attempts are escaped; (4) Use JSTL <c:out> tag or explicit escaping functions instead of <%= %>; (5) Implement Content Security Policy headers; (6) Test stored XSS by inserting malicious data in database and verifying it's escaped on display.
**File:** All JSP files
**Line(s):** All dynamic output lines (<%=...%>) throughout files
**Untested function:** HTML output escaping, XSS prevention, content sanitization

### P76-F15: No edge case tests for empty data scenarios
**Severity:** MEDIUM
**Description:** None of the JSP files have tests verifying correct behavior when query results are empty: (1) Empty ArrayList from filter methods; (2) Zero vehicles/drivers/sessions; (3) No pre-op checks in date range; (4) No restricted access usage. The code may display empty tables, divide-by-zero errors in averages, or confusing UI when no data exists.
**Fix:** Implement empty data scenario tests: (1) Test each report with zero results from database; (2) Verify user-friendly message is displayed when no data exists; (3) Test average calculations handle division by zero; (4) Test table rendering with empty data; (5) Test total row calculations with zero records; (6) Verify print functionality doesn't break with empty data; (7) Consider adding "No data found" messages instead of empty tables.
**File:** All JSP files
**Line(s):** All ArrayList iteration and calculation sections
**Untested function:** Empty data handling, zero-record scenarios, average calculations with no data
