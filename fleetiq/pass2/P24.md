**Severity:** HIGH
**Description:** The frm_new_location.jsp file contains client-side validation for location name and timezone format (lines 19-28, 55-79) but has no automated tests. Validation checks for null/empty location names and GMT timezone format (+/-HH:MM) with constraints (max ±12:00, exactly 12:00 must be 12:00). Test edge cases: empty strings, whitespace-only input, invalid timezone formats, boundary values (±12:00, ±12:01), and special characters in location names.
