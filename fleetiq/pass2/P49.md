## Findings

### P49-F01: No null safety on request.getParameter() calls
**Severity:** CRITICAL
**Description:** Lines 65-66 in report_head_with_filter.jsp and lines 66-67 in report_head_with_filter_keyword.jsp call request.getParameter("mnm").toString() and request.getParameter("sub").toString() without null checks. If these parameters are not present in the request, this will throw NullPointerException, causing application crash. This code path has no test coverage to detect this critical failure scenario.
**Fix:** Add null checks before calling toString() or provide default values. Example: String mnm = request.getParameter("mnm"); if(mnm == null) mnm = "default"; Create unit tests to verify null parameter handling returns appropriate error or default value instead of crashing.
**File:** pages/admin/reports/inc/report_head_with_filter.jsp, pages/admin/reports/inc/report_head_with_filter_keyword.jsp
**Line(s):** 65-66, 66-67
**Untested function:** request.getParameter() null case

### P49-F02: No test coverage for session attribute null checks
**Severity:** CRITICAL
**Description:** Lines 49-51 in both files retrieve session attributes "user_cd", "dealer_id", and line 53 retrieves "permision" array without null checks. If session expires or attributes are not set, this causes NullPointerException. Line 50 attempts null check for "dealer_id" but uses dangerous pattern (session.getAttribute("dealer_id") == null ? 0:Integer.parseInt(...)) that can still throw NumberFormatException if attribute is not null but not a valid integer. No tests verify session expiration scenarios or malformed session data.
**Fix:** Add comprehensive null checks for all session attributes with appropriate error handling. Wrap Integer.parseInt() in try-catch to handle NumberFormatException. Create integration tests that simulate expired sessions, missing session attributes, and malformed session data to verify graceful degradation.
**File:** pages/admin/reports/inc/report_head_with_filter.jsp, pages/admin/reports/inc/report_head_with_filter_keyword.jsp
**Line(s):** 49-51, 53
**Untested function:** Session expiration and invalid session data handling

### P49-F03: No test coverage for Integer.parseInt(form_cd) injection vulnerability
**Severity:** CRITICAL
**Description:** Lines 156, 159, 162, 166 in report_head_with_filter.jsp and lines 166, 188, 193 in report_head_with_filter_keyword.jsp call Integer.parseInt(form_cd) to access permision array without validating that form_cd is numeric or within valid bounds.
