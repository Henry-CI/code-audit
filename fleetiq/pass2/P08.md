**Severity:** CRITICAL
**Description:** The JSP directly outputs request parameters using `<%=request.getParameter("mnm").toString()%>` and `<%=request.getParameter("sub").toString()%>` at lines 3-4 without HTML encoding. There are no tests to verify that these parameters are properly sanitized or encoded, leaving the application vulnerable to Cross-Site Scripting (XSS) attacks. An attacker could inject malicious JavaScript through the `mnm` or `sub` parameters.
**Fix:** Create integration tests that verify: (1) XSS payloads in `mnm` and `sub` parameters are properly encoded before output; (2) Special characters like `<`, `>`, `"`, `'`, and `&` are escaped; (3) Add unit tests for any encoding/sanitization utility functions used. Implement proper HTML encoding using JSTL `<c:out>` or `fn:escapeXml()` instead of direct scriptlet output.
