### P03-F01: Missing Session Validation Tests
**Severity:** CRITICAL
**Description:** All 10 JSP files retrieve session attributes (access_level, access_cust, access_site, access_dept) without null checks before use. If session.getAttribute() returns null, the code will throw NullPointerException when calling .equalsIgnoreCase("") on access_level. There are no unit tests to verify session validation, null handling, or the behavior when session attributes are missing or expired.
**Fix:** Create integration tests that verify behavior when session attributes are null, empty, or invalid. Add defensive null checks in the JSP code before accessing session attributes. Test scenarios: missing session, expired session, null access_level, empty access_level.
**File:** All 10 JSP files
**Line(s):** xlsx_driver_impact_report.jsp:36-41, xlsx_dyn_driver_report.jsp:39-44, xlsx_dyn_unit_report.jsp:40-45, xlsx_impact_meter_report.jsp:38-43, xlsx_impact_report.jsp:37-42, xlsx_key_hour_util_report.jsp:34-39, xlsx_preop_chk_fail_report.jsp:39-44, xlsx_preop_chk_report.jsp:39-44, xlsx_restricted_access_usage_report.jsp:40-45, xlsx_seat_hour_util_report.jsp:34-39
**Untested function:** session.getAttribute() null handling

### P03-F02: Missing Input Validation Tests for Request Parameters
**Severity:** HIGH

Parameters like user_cd, loc_cd, dept_cd, search_crit, form_cd are passed directly to database queries (via filter beans) without validation. No tests exist for SQL injection prevention, XSS prevention, or parameter tampering.
**Fix:** Create comprehensive input validation tests covering: null inputs, empty strings, special characters, SQL injection attempts, XSS payloads, excessively long strings, boundary values for numeric parameters. Implement input sanitization before using parameters in database operations.
**File:** All 10 JSP files
**Line(s):** Lines 19-36 (varies by file)
**Untested function:** request.getParameter() validation and sanitization

### P03-F03: Missing Access Control Authorization Tests
**Severity:** CRITICAL
**Description:** All files implement access level control by setting filter properties (access_level, access_cust, access_site, access_dept) but there are no tests verifying that unauthorized users cannot access reports or that access controls are properly enforced. The code defaults access_level to "5" when empty, but there are no tests ensuring this default is secure or appropriate. No tests verify that users with insufficient access_level cannot view data from other customers, sites, or departments.
**Fix:** Create authorization tests for each access level (1-5). Test horizontal privilege escalation (user accessing another customer's data), vertical privilege escalation (low-level user accessing admin reports), and boundary conditions. Verify that the default access_level of "5" is appropriate and tested.
**File:** All 10 JSP files
**Line(s):** Lines 41-49 (varies by file)
**Untested function:** Access level enforcement and authorization logic

### P03-F04: Missing File System Path Traversal Tests
**Severity:** HIGH
**Description:** All JSP files construct file system paths using string manipulation (getProtectionDomain().getCodeSource().getLocation(), substring(), replace(), concatenation). The directory path construction on lines 79-83 (and similar) uses relative path traversal ("/../../../../../../../excelrpt/") which is vulnerable to path traversal attacks. There are no tests verifying that attackers cannot manipulate the path to access files outside the intended directory.
**Fix:** Create tests for path traversal vulnerabilities. Test with inputs containing "..", absolute paths, null bytes, and special characters. Use secure path APIs (Path.normalize(), Files.exists()) instead of string concatenation. Validate that generated paths remain within the expected directory boundaries.
**File:** All 10 JSP files
**Line(s):** xlsx_driver_impact_report.jsp:79-83, xlsx_dyn_driver_report.jsp:92-97, xlsx_dyn_unit_report.jsp:92-96, xlsx_impact_meter_report.jsp:80-85, xlsx_impact_report.jsp:78-83, xlsx_key_hour_util_report.jsp:90-95, xlsx_preop_chk_fail_report.jsp:80-85, xlsx_preop_chk_report.jsp:84-89, xlsx_restricted_access_usage_report.jsp:71-75, xlsx_seat_hour_util_report.jsp:90-95

### P03-F05: Missing File Download Security Tests
**Severity:** HIGH
**Description:** All files implement file download functionality (lines 91-111 and similar) with try-catch blocks that only print stack traces on errors. There are no tests for: file not found scenarios, permission errors, large file handling, concurrent download attempts, or resource exhaustion. Implement try-with-resources for automatic resource management.

### P03-F06: Missing Database Query Injection Tests
