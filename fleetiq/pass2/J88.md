### J88-F01: LockOutDAO has zero test coverage
**Severity:** CRITICAL
**Description:** The LockOutDAO class has no unit tests covering the getLockOutData method. This DAO performs SQL queries with string concatenation for building the unit list, handles database connections, processes result sets with complex type mapping logic (reason codes 252, 253), and has exception handling that could mask errors. **Fix:** Create LockOutDAOTest.java with test cases covering: (1) successful data retrieval with valid parameters, (2) empty result set handling, (3) null/empty cust_cd and loc_cd parameters, (4) database connection failure scenarios, (5) SQL exception handling, (6) proper resource cleanup verification, (7) reason code mapping for values 252, 253, and other values, (8) null values in ResultSet columns.
