### P12-F01: No authentication tests for session validation
**Severity:** CRITICAL
**Description:** All three JSP files access session attributes (user_cd, access_level, access_cust, access_site, access_dept) to enforce access control, but there are no tests verifying that unauthenticated users are properly denied access, or that expired sessions result in appropriate error handling. The include "_expire.jsp" (line 1 in all files) is intended to handle session expiration but has no test coverage.
**Fix:** Create integration tests that verify: (1) requests without valid sessions are rejected, (2) expired sessions trigger appropriate redirects or errors, (3) session attributes are properly validated before data export is allowed.
**File:** exporter/checklist.jsp, exporter/users.jsp, exporter/vehicles.jsp
**Line(s):** checklist.jsp:1,22; users.jsp:1,20-23; vehicles.jsp:1,20-23
**Untested function:** Session validation logic in _expire.jsp include

### P12-F02: No authorization tests for access level enforcement
**Severity:** CRITICAL
**Description:** users.jsp (lines 20-29) and vehicles.jsp (lines 20-29) retrieve and parse access_level from session, defaulting to "5" if empty. There are no tests verifying: (1) that users with insufficient access levels cannot export data, (2) that the default value "5" is appropriate and secure, (3) that access_cust, access_site, and access_dept session attributes properly restrict data visibility.
**Fix:** Create tests that verify users with different access levels only see data they're authorized to access. Test edge cases: null access_level, empty string, invalid integers, extremely high/low values. Verify that access_cust/site/dept filters are properly applied to prevent horizontal privilege escalation.
**File:** exporter/users.jsp, exporter/vehicles.jsp
**Line(s):** users.jsp:20-29; vehicles.jsp:20-29
**Untested function:** Access level parsing and validation

### P12-F03: No input validation tests for SQL injection via request parameters
**Severity:** CRITICAL
**Description:** All three JSP files retrieve multiple request parameters and pass them directly to Databean filter objects which execute database queries. No tests verify that malicious input (SQL injection payloads, special characters, XSS attempts) is properly sanitized. Parameters include: vehicle_cd, div_cd, module_cd, form_cd, customer, division, location, department, model, status, search. The parameters are passed to filter.query() methods without visible sanitization.
**Fix:** Create tests with malicious payloads: SQL injection strings (' OR '1'='1, UNION SELECT, etc.), XSS payloads (<script>alert(1)</script>), null values, extremely long strings (>10000 chars), special characters. Verify that Databean_getter and Databean_customer properly sanitize/parameterize all inputs before database queries.
**File:** exporter/checklist.jsp, exporter/users.jsp, exporter/vehicles.jsp
**Line(s):** checklist.jsp:14-18,29; users.jsp:12-17,45; vehicles.jsp:14-17,44
**Untested function:** request.getParameter() followed by filter.query()

### P12-F06: No tests for CSV injection vulnerabilities
**Severity:** HIGH
**Description:** All three JSP files output user-controlled data directly to CSV format without escaping formula characters (=, +, -, @). Attackers could inject CSV formulas that execute when the file is opened in Excel/Sheets. For example, if a user's name is "=1+1" or "=cmd|'/c calc'!A1", it would be executed as a formula. Lines where data is written: checklist.jsp:65-74, users.jsp:102-121, vehicles.jsp:114-128.
**Fix:** Create tests with malicious CSV payloads in all user-controllable fields: names starting with =, +, -, @, tab characters, newlines, quotes. Verify that output properly escapes or strips formula characters. Implement CSV escaping: prepend single quote to cells starting with =+-@, or use a CSV library that handles escaping.
**File:** exporter/checklist.jsp, exporter/users.jsp, exporter/vehicles.jsp
**Line(s):** checklist.jsp:65-74; users.jsp:102-121; vehicles.jsp:114-128
**Untested function:** CSV data output with user-controlled values
