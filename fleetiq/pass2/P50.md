### P50-F01: No test coverage for SQL injection vulnerability in mail.Ename()
**Severity:** CRITICAL
**Description:** The `mail.Ename(String userid)` method (mail.java:32-98) constructs a SQL query using unsanitized string concatenation (line 54: `"select ... where \"USER_CD\"='"+userid+"'"`). This method is called from mail_report.jsp line 22 with session attribute `user_cd`, which could be manipulated. No unit tests exist to verify SQL injection prevention, parameterized queries, or input validation.
**Fix:** Create unit tests for `mail.Ename()` that verify: (1) proper parameterized query usage or input sanitization, (2) SQL injection attempts are blocked (test with payloads like `' OR '1'='1`, `'; DROP TABLE--`), (3) proper error handling for malformed input. Refactor the method to use PreparedStatement instead of string concatenation.
**File:** pages/admin/reports/mod/mail_report.jsp
**Line(s):** 22
**Untested function:** com.torrent.surat.fms6.util.mail.Ename(String)
