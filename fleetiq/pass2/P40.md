## Findings

### P40-F01: Complete absence of automated tests for driver_licence_expiry_report.jsp
**Severity:** CRITICAL
**Description:** No automated tests exist for driver_licence_expiry_report.jsp, which handles security-sensitive access control and performs unsafe substring operations on session attributes (lines 37-39) without null or length validation. The access level parsing (line 30) uses Integer.parseInt() without exception handling, creating potential for application crashes. SQL injection vector exists via op_code parameter (line 18).
**Fix:** Create comprehensive test suite covering: (1) Session attribute null/empty cases, (2) substring operations with various string lengths including empty strings, (3) Integer.parseInt with invalid/null values, (4) Access level boundary conditions (levels 1-5), (5) ArrayList bounds checking for all customer/site/dept operations, (6) XSS injection in filter parameters.
**File:** pages/admin/reports/driver_licence_expiry_report.jsp
**Line(s):** 30, 37-39, 65-82, 121-122
**Untested function:** All scriptlet logic, session validation, access control, filter initialization

**Description:** driver_proficiency_report.jsp lacks any test coverage despite handling user input (search_crit, do_list) without sanitization and constructing URLs via string concatenation (lines 91-93). Direct parameter concatenation in AJAX URLs creates XSS vulnerability. SQL injection possible through report_code variable (line 67). No validation exists for customer selection before processing (lines 73-81).
**Fix:** Implement tests for: (1) Input sanitization on search_crit and do_list parameters, (2) URL encoding in email/export functionality, (3) Customer validation error paths, (4) AJAX request construction with malicious input, (5) SQL injection attempts via report_code, (6) Null/empty dealer, cust_cd, loc_cd handling.
**File:** pages/admin/reports/driver_proficiency_report.jsp
