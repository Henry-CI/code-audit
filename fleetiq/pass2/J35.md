## Findings

### J35-F01: No test coverage for file path injection vulnerability in Frm_excel constructor
**Severity:** CRITICAL
**Description:** Multiple constructors in Frm_excel accept user-controlled file paths (docRoot, fileName) without validation and use them to construct file system paths. The fileName parameter is processed with replace(" ", "") but has no path traversal validation. An attacker could inject "../" sequences or absolute paths to read/write files outside intended directories. Methods like getExportDir() (line 833) concatenate paths unsafely.
**Fix:** Add unit tests that verify: (1) Path traversal sequences are rejected or sanitized, (2) Absolute paths are rejected, (3) Only whitelisted characters are allowed in fileName, (4) Constructed paths stay within expected directory boundaries. Test with inputs like: "../../../etc/passwd", "../../malicious.xlsx", "/etc/shadow.xlsx", "file\0name.xlsx".
**File:** WEB-INF/src/com/torrent/surat/fms6/excel/Frm_excel.java
**Line(s):** 62-139, 833-845
**Untested function:** Frm_excel (all 9 constructors), getExportDir
