### P36-F10: No test coverage for XSS vulnerabilities in JSP output

**Description:** All four JSP files output user-controlled or database-retrieved data directly into HTML without visible escaping (e.g., site_hours.jsp:227 outputs message, view.jsp:82-98 outputs customer data fields). There are no tests verifying that malicious HTML/JavaScript in database fields cannot be executed in the browser. If an attacker can inject script tags into customer names, site names, or configuration fields, they could perform XSS attacks against other users viewing the data.
**Fix:** Create XSS security tests: (1) Test that HTML tags in customer names are escaped (e.g., "<script>alert('xss')</script>" in customer name should render as text, not execute), (2) Test that JavaScript event handlers in input fields are escaped, (3) Test that all output variables use proper escaping: <%=Encode.forHtml(value)%> or <c:out value="${value}"/>, (4) Review all <%= %> expressions and ensure they escape output, especially for: message, customer names, site names, department names, configuration fields, (5) Use automated XSS scanning tools against the pages, (6) Test that user-supplied data in error messages is escaped.
**File:** pages/admin/customer/site_hours.jsp, pages/admin/customer/site_setup_config.jsp, pages/admin/customer/update_site_hours.jsp, pages/admin/customer/view.jsp
