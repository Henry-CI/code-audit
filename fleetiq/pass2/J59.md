### J59-F07: Static mutable state in multi-threaded environment
**Severity:** CRITICAL
**Description:** ImpactReportBean declares 7 static mutable fields (machineId, model, driverName, cardNo, dtImpact, severityLvl, fileName at lines 7-13). In a web application, static fields are shared across ALL user requests and threads. When one user's request calls setMachineId(), it overwrites the value for all concurrent users. This creates race conditions where User A's report data can be corrupted by User B's concurrent request. There are no tests demonstrating this concurrency bug, and the bean is used in 17+ production files including critical impact and driver abuse reports. This can lead to users seeing other users' sensitive data or corrupted reports.
**Fix:** Create concurrency tests in ImpactReportBeanTest.java that demonstrate the data corruption issue by simulating multiple concurrent threads calling static setters and getters. Tests should verify that when Thread 1 sets machineId="A" and Thread 2 sets machineId="B" concurrently, both threads can retrieve different values. This will demonstrate the bug. Then refactor the class to use instance fields instead of static fields, or implement ThreadLocal storage if class-level state is required. Rerun tests to verify thread-safety after refactoring.
