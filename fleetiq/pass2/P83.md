### P83-F01: Complete absence of test coverage for access control logic
**Severity:** CRITICAL
**Description:** All 10 JSP files implement session-based access control logic (access_level, access_cust, access_site, access_dept) with NO test coverage. The access level validation (lines 28-32 in multiple files) defaults to "5" when access_level is empty, which is a security-critical default that could grant unintended access. This logic executes on every page load and controls data visibility but has zero automated tests to verify it works correctly or handles edge cases (null session, missing attributes, invalid levels).
**Fix:** Create integration tests that verify: (1) access control restricts data based on access_level values 1-5, (2) empty/null access_level correctly defaults to "5", (3) invalid access levels are rejected, (4) session attribute tampering is prevented, (5) cross-customer/site/dept data leakage is prevented.
**File:** reports/rpt_seat_hour_util.jsp, rpt_serv_maintenance.jsp, rpt_subscription.jsp, rpt_unit_unlock.jsp, rpt_unit_unlock_impact.jsp, rpt_unit_unlock_question.jsp, rpt_user_email_summary.jsp, rpt_user_summary.jsp

**Severity:** CRITICAL
**Description:** All JSP files pass unsanitized request parameters (gp_cd, user_cd, loc_cd, dept_cd, st_dt, to_dt, form_cd) directly to filter bean setters, which are then used in database queries via filter.init(). No test coverage exists to verify that the Databean_report bean properly sanitizes these inputs before building SQL queries. The only parameter validation is null-coalescing to empty string or default values, providing no protection against SQL injection attacks.
**Fix:** Create comprehensive test suite for Databean_report.init() that verifies: (1) SQL injection attempts in all parameters are properly escaped/rejected, (2) special characters in parameters don't break queries, (3) malformed date inputs are rejected, (4) boundary values are handled safely, (5) parameterized queries or prepared statements are used throughout.
**File:** reports/rpt_seat_hour_util.jsp, rpt_serv_maintenance.jsp, rpt_subscription.jsp, rpt_unit_unlock.jsp, rpt_unit_unlock_impact.jsp, rpt_unit_unlock_question.jsp, rpt_user_email_summary.jsp, rpt_user_summary.jsp
