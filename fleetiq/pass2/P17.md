## Findings

### P17-F01: No test coverage for redirect functionality
**Severity:** MEDIUM
**Description:** The index.jsp file implements a dual redirect mechanism (meta refresh and JavaScript onLoad) to navigate users to pages/login.jsp. There are no automated tests verifying that this redirect functions correctly, that it redirects to the intended target, or that it cannot be manipulated for open redirect attacks. The lack of testing means regression issues could go undetected.
**Fix:** Create integration tests (e.g., using Selenium WebDriver or similar) to verify: (1) the page successfully redirects to pages/login.jsp, (2) the redirect cannot be manipulated via query parameters or other inputs, (3) the redirect completes within expected timeframe. Add negative tests for invalid target paths.
**File:** index.jsp
**Line(s):** 4, 7-11, 14
**Untested function:** send() JavaScript function, meta refresh redirect

**Fix:** Implement HTML validation tests using tools like HTML Tidy, W3C Validator, or jsoup to programmatically validate page structure. Include tests that verify the page renders correctly in different browsers.

### P17-F03: No test for JavaScript syntax error
**Severity:** MEDIUM
**Description:** Line 10 contains a JavaScript syntax error: `location.href(url)` incorrectly attempts to invoke location.href as a function when it should be a property assignment (`location.href = url`). This error would cause the JavaScript redirect to fail, but there are no tests to catch this. The meta refresh provides a fallback, but the JavaScript error still represents untested failure path.
**Fix:** Create JavaScript unit tests using frameworks like Jest, Jasmine, or QUnit to test the send() function. Add integration tests that verify JavaScript execution and error handling. Implement ESLint or similar static analysis in the build pipeline to catch syntax errors before deployment.
**File:** index.jsp
**Line(s):** 10
**Untested function:** send() JavaScript function
