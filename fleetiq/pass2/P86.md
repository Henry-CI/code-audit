### P86-F01: Complete Absence of Automated Tests for Security-Critical JSP Forms
**Severity:** CRITICAL
**Description:** All 10 JSP files in the security module lack any form of automated testing (unit, integration, or end-to-end). These files handle highly sensitive operations including password changes, access rights management, form/module administration, and mail group configuration. The entire repository contains zero actual test files (EncryptTest.java is a decompiled utility, not a test). This creates unmaintained, unverified security controls with no regression protection.
**Fix:** Implement comprehensive test coverage using Selenium WebDriver for UI tests and JUnit/Mockito for servlet/bean layer tests. Minimum required: (1) Selenium tests for each JSP covering happy path, validation failures, and XSS/SQL injection attempts; (2) JUnit tests for Frm_security servlet methods (save_form, change_password, save_perm, etc.) with mocked database connections; (3) Integration tests for AJAX endpoints (get_form_data.jsp) validating XML output and error handling; (4) Security-focused tests for CSRF, session hijacking, and authentication bypass scenarios.
**File:** All 10 files
**Line(s):** N/A (architectural gap)
**Untested function:** All functions across all files

### P86-F02: Untested Password Change Validation Logic
**Severity:** CRITICAL
**Description:** Both auto_change_pass.jsp and frm_change_pass.jsp implement client-side password validation with zero server-side test coverage. The saveform() functions perform critical security checks (password match, null/empty validation) that are never tested. auto_change_pass.jsp includes complex password strength regex (strongRegex, mediumRegex, enoughRegex) that is completely untested. A bug in these validations could allow weak passwords or bypass password policies entirely.
**Fix:** Create Selenium tests validating: (1) Empty password rejection; (2) Password mismatch detection; (3) Password strength levels (weak/good/strong) using test vectors; (4) Boundary cases (5, 6, 7 character passwords); (5) Special character handling. Create JUnit tests for server-side password validation in Frm_security.change_password() method verifying BCrypt hashing, password history checks, and database transaction rollback on failure.
**File:** security/auto_change_pass.jsp, security/frm_change_pass.jsp
**Line(s):** auto_change_pass.jsp:14-48, 218-235; frm_change_pass.jsp:15-49
**Untested function:** saveform(), passwordChanged()

### P86-F03: Untested Access Rights Permission Management
**Severity:** CRITICAL
**Description:** frm_access_rights.jsp implements a complex permission system (view/edit/delete/print) for multiple forms with zero test coverage. The saveform() function (lines 16-117) maps checkbox states to hidden fields using conditional logic for count==1 vs count>1 scenarios. Bugs in this logic could grant unintended elevated privileges or silently fail to save permission denials.
