**Description:** The file print_impact_meter_report.jsp has no automated tests. This JSP processes security-sensitive session data (access_level, access_cust, access_site, access_dept) and renders impact sensor data with severity classifications. Critical security logic at line 28 defaults access_level to "5" if empty, which could grant unintended access. No tests verify: (1) session attribute null handling, (2) access level boundary validation, (3) nested ArrayList iteration safety, (4) severity level color coding accuracy, (5) XSS protection in output rendering.
**Fix:** Create integration test suite covering: session attribute validation, access control enforcement, null/empty parameter handling, nested data structure iteration without IndexOutOfBoundsException, severity level logic verification, and output escaping.
**File:** au_print/print_impact_meter_report.jsp
**Line(s):** 4-69, 93-168

### P06-F02: Null pointer vulnerabilities in session access (impact meter report)
**Severity:** CRITICAL
**Description:** Lines 23-26 retrieve session attributes without null checks. If session is invalid or attributes are missing, lines 28-32 will throw NullPointerException on access_level.equalsIgnoreCase(""). The parseInt at line 32 is never used (variable 'al' is assigned but never referenced), suggesting incomplete access control implementation. No tests verify session state.
**Fix:** Add null checks before equalsIgnoreCase(), validate session existence, remove dead code (variable 'al'), add tests for invalid session scenarios, expired sessions, and missing attributes.
**File:** au_print/print_impact_meter_report.jsp
**Line(s):** 23-32
**Untested function:** Session attribute retrieval and access level validation

### P06-F03: ArrayIndexOutOfBoundsException risk in nested loops (impact meter report)
**Severity:** HIGH
**Description:** Lines 93-168 contain nested loops accessing parallel ArrayLists (Vdriver, Vcard_no, Vimp_tm, Vimp_ene, meterDetails, Vtmp_imp_data) without size validation. Line 104-107 assumes tmpMeter has same size as dnm, line 108 checks dnm.size() but doesn't validate other lists. If ArrayLists are misaligned, runtime exception occurs. No tests verify data structure consistency.
**Fix:** Add size validation before all ArrayList accesses, add defensive null checks, create tests with mismatched ArrayList sizes, empty lists, and null elements.
**File:** au_print/print_impact_meter_report.jsp
**Line(s):** 93-168
**Untested function:** Nested ArrayList iteration and parallel data structure access

### P06-F04: XSS vulnerabilities in unescaped output (impact meter report)
**Severity:** CRITICAL
**Description:** Lines 112-160 output ArrayList values directly to HTML without escaping: <%=Vhire_no.get(i)%>, <%=Vmodel_no.get(i)%>, <%=dnm.get(j)%>, etc. If database contains malicious scripts in hire_no, model_no, driver names, or card numbers, XSS attack executes in user browser. No tests verify output sanitization.
**Fix:** Use JSTL <c:out> tags or explicit escaping for all dynamic content. Add tests with XSS payloads (<script>, <img onerror>, javascript:) in all output fields.
**File:** au_print/print_impact_meter_report.jsp
